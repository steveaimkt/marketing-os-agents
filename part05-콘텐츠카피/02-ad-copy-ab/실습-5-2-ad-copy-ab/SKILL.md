# Part 5 / 5-2 · 광고 카피 10패턴 A/B 실습 : 소개 → 실행 → 활용

> 핵심 메시지 1줄을 **10가지 설득 패턴으로 변형하고 6축으로 자동 채점**하는 에이전트를 가동한다.
> 핵심 학습 목표 : ① 10패턴 변형 ② 에이전트가 스킬(quality-review-6axis)을 호출하는 구조 ③ 점수 기반 선별.
> ⛔ 자동 실행 금지. 단계마다 사용자 답을 받은 뒤에만 다음으로.

## 진행 흐름 (3 단계)

```
1단계  소개   에이전트가 하는 일 + 연결된 소스                   ⏸ 답 대기
2단계  실행   사전 점검 → 작동 구조 → 데이터 준비 → 실제 실행   ⏸ 답 대기
3단계  활용   내 업무 적용법 + 산출물 + 다음 클립               ⏸ 선택
```

---

## 1단계 · 소개

```
✍️ 광고 카피 10패턴 A/B 에이전트를 소개합니다.

본 에이전트가 하는 일 :
  "카피 1개를 짜는 게 아니라, 10가지 설득 구조를 한 번에 깔고 6축 점수로 고른다"

⏱ Before vs After
  · Before : 카피 5안 짜는 데 1시간 + 어떤 게 나은지 감으로 선택
  · After  : 10패턴 + 6축 점수 + TOP 3 추천 = 약 3분, 선택 근거가 숫자로 남음

🔌 연결 소스 (실제 의존만)
  ┌─ Skill : ad-copy-ab               ← 핵심 메시지 1개 → 10패턴 변형
  ├─ Skill : quality-review-6axis     ← 10종 전부 6축 자동 채점 (에이전트가 호출)
  ├─ Notion "Brand Voice" 페이지       ← 톤 3 + 금기어 (5-1 산출물, 생성 시 자동 준수)
  ├─ Google Sheets MCP                ← 카피 10종 + 점수 적재
  └─ Discord webhook                  ← TOP 3 발송

💡 핵심 포인트
  · 에이전트(ad-copy-ab)가 스킬(quality-review-6axis)을 도구처럼 호출하는
    "에이전트 → 스킬" 체이닝이 이번 클립의 학습 핵심.
  · 같은 채점 스킬을 landing-copy · email-newsletter 도 재사용 (DRY).

👉 2단계(실행)로 넘어갈까요?
```

📐 **스킬? 에이전트? — 이 클립은 둘 다 나온다**

```
· 카피 10패턴 '생성'  = 입력만 바뀌는 순수 변환 → 🧩 스킬 ad-copy-ab
· 6축 '채점'          = 또 다른 순수 변환        → 🧩 스킬 quality-review-6axis
· 그 둘을 불러 Sheets 적재·Discord 발송 = 도구·산출 있음 → 🤖 에이전트가 감쌈
⇒ "에이전트가 스킬 2개를 도구처럼 부른다" 가 이번 학습 핵심
```

⏸ 답 받고 2단계로.

### 2-3 · 실제 실행

```
▶ 에이전트 워크플로 단계별 실행:

  1. Brand Voice 로드
  2. 10패턴 생성                          → 패턴명과 함께 10종 표시 (시연 하이라이트)
       글자수 제한 준수 여부 함께 표시
  3. quality-review-6axis 호출            → 10종 × 6축 채점표 (에이전트 → 스킬 체이닝)
  4. Google Sheets 적재                   → 10행 + 점수
  5. TOP 3 선별                           → 디스코드 embed (평균 점수 + 축별 강점 한 줄)
  6. 품질 게이트 판정                     → 전체 평균 표시 · 3.5 미만 패턴은 수정 제안 1건

피드백 루프 1회: "TOP 1 을 더 짧게" 같은 한 줄 수정 → 재생성 + 재채점.
```

### 오래 쓰는 법 (적용 후 정착)

```
🎯 내 업무에 적용하는 법

  1. 신제품 런칭 카피 : 핵심 메시지 1줄만 정하면 10패턴이 동시에 깔려

  2. 채널별 변형 : 같은 메시지로 채널·목표·글자수만 바꿔 재실행 → 채널별 최적안 확보.
  3. 채점 기준 공유 : quality-review-6axis 한 스킬을 5-3 랜딩·3-1 뉴스레터도 함께 호출 →

  4. 품질 게이트 운영 : REVISION_RECOMMENDED(평균 3.5 미만) 와 BLOCKED(규제 2 이하)를

  [ ] REVISION_RECOMMENDED 와 BLOCKED 의 차이를 이해
```
