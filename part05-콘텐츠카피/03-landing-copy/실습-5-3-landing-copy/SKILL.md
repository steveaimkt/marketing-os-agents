# Part 5 / 5-3 · 상세페이지 카피 에이전트 실습 : 소개 → 실행 → 활용

> 제품 ID 하나로 **상세페이지 24섹션을 생성하되, 화장품법 금기어를 자동 치환**하는 에이전트를 가동한다.
> 핵심 학습 목표 : ① 선행 산출물 자동 로드 (VoC·레퍼런스) ② 페인 섹션 = 고객의 언어 ③ 규제 게이트.
> ⛔ 자동 실행 금지. 단계마다 사용자 답을 받은 뒤에만 다음으로.

## 진행 흐름 (3 단계)

```
1단계  소개   하는 일 + Before/After + 연결 소스          ⏸ 답 대기
2단계  실행   작동 구조 → 데이터 준비 → 24섹션 + 규제 치환   ⏸ 답 대기
3단계  활용   내 업무 적용법 + 산출물 + 다음 클립           ⏸ 선택
```

---

## 1단계 · 소개

```
📄 상세페이지 카피 에이전트 — 무엇을 하는가

이 에이전트는 product_id 하나만 받아, 선행 에이전트들이 만든 산출물을 자동으로
끌어와 상세페이지 24섹션 초안을 만든다. 핵심은 "잘 쓰는 것보다 먼저, 법에
걸리지 않게. 그 다음 고객의 언어로".

⏱ Before vs After
  · Before : 상세페이지 카피 외주 3~7일 + 심의 리스크는 사람 눈에 의존
  · After  : 24섹션 초안 10분 + 금기어 자동 검출·치환 로그까지
```

🔌 연결 소스 (이 에이전트가 실제로 의존하는 것만)

```
[product_id : TONER-V2]
   ├─ Skill : landing-copy          ← 24섹션 구조 + 규제 치환 사전 정의
   ├─ Skill : brand-voice           ← 톤·금기어·대체 표현 로드
   ├─ Notion "Brand Voice" (5-1)    ← 톤 3 + 금기어 표 + 페르소나 2
   ├─ VoC TOP 5 (4-4 산출물)        ← 페인 섹션의 고객 언어 원문
   └─ 광고 레퍼런스 (4-3)           ← USP·CTA 패턴 참고
        ↓
   Notion 상세페이지 페이지 (24섹션 + 규제 치환 로그)
```

💡 핵심 포인트
  · 입력이 product_id 하나 : 나머지는 전부 선행 에이전트 산출물 자동 로드 → 파이프라인이 깊어질수록 사람 입력은 줄어든다.
  · 페인 섹션은 "마케터의 추측"이 아니라 VoC 의 고객 문장을 그대로 인용한다.
  · 규제 게이트가 워크플로 마지막이 아니라 생성 중간에 있다 (사후 검열이 아닌 사전 차단).

📐 **스킬? 에이전트?** → 스킬(landing-copy·brand-voice)을 불러 Notion 24섹션 페이지 생성(도구·산출) → 🤖 **에이전트**. 만드는 법은 3-1의 **에이전트 5요소** 골격 그대로(스킬 2개를 tools로 끼움).

⏸ "실행해 볼까요?" — 답 받고 2단계로.

### 2-1 · 작동 구조

```
▶ 에이전트 워크플로
  [product_id : TONER-V2]
    → 자동 로드 : Brand Voice (5-1) + VoC TOP 5 (4-4) + 광고 레퍼런스 (4-3)
    → 24섹션 생성 :
       1~11   : 메인 USP · 페인 3각도 (VoC 고객 언어 그대로) · 솔루션 · 성분 · 임상 ·
                차별점 · 사용법 · 페르소나 · FAQ · 리뷰
       12~24  : 혜택 반복 + CTA 12종 (다변량 테스트용)
    → 규제 게이트 : 금기어 검출 → 대체 표현 자동 치환 + 치환 로그
    → 임상 수치 인용 규칙 : 출처·n수 병기 강제
    → Notion 페이지 저장

🛡 규제 게이트가 BLOCKED 를 내면 : 페이지 생성을 멈추고 위반 목록을 보고 (생성 강행 불가)
```

  [B] 실데이터 모드
      본인 제품 스펙 (성분·가격·타겟) 을 직접 입력. 본인 업종의 규제 체계 확인 동반.

`sample-data/` 3종 로드 + 확인 (모드와 무관하게 구조는 동일):

1. **product-spec-toner-v2.md** : 결로아 포어 토너 V2 (250ml / 24,000원) · 성분 4종 (PHA 5% · 어성초 21% · 나이아신아마이드 2% · 판테놀 1%) · 임상 3건 (모공 개선 체감 78% n=42 등) · FAQ 소재
2. **voc-top5.json** : 페인 5건 + 고객 표현 원문 ("펌프가 한 번에 너무 많이 나와요" · "입구가 좁아서 화장솜에 적시기 불편" 등) — 4-4 분석 결과와 동일 수치
3. **brand-voice.md** : 금기어 표 (모공 축소 → 모공 케어 / 재생·치료·주름 개선 사용 불가) — 치환 사전으로 사용

### 2-3 · 실제 실행

```
▶ 에이전트 워크플로 단계별 실행:

  1. 선행 산출물 3종 자동 로드 확인
  2. 섹션 1~11 생성                       → 구조 미리보기 (페인 섹션에 고객 문장 인용 하이라이트)
  3. 규제 게이트 (시연 하이라이트):
       초안 검출 : "모공 축소" 1건 · "재생" 1건 발견
       → 자동 치환 : 모공 축소 → 모공 케어 / 재생 → (제거 + 표현 재작성)
       → 치환 로그 표 출력 (검출어 · 위치 · 대체어 · 근거 조항)
  4. 임상 인용 검사                       → 수치 3건 모두 출처·n수 병기 확인
  5. 섹션 12~24 (CTA 12종) 생성
  6. Notion 페이지 저장                   → URL
```

## 3단계 · 활용 — 지금, 내 업무로

### 적용 게이트 (능동 질문 ⏸)

```
🚀 그럼, 지금부터 내 업무에 적용해볼까요?

   ① 어떤 제품인가요?
   ② 상세페이지에 꼭 들어갈 핵심 정보는?  (성분·효능·가격·FAQ 등)
   ③ 업종 규제 사전을 쓸까요?            (화장품·건기식 등 / 일반)
```

→ 답을 받으면 : 24섹션 상세 초안 + 규제 치환 로그(검출어·대체어).
→ "건너뛰기" 시 아래 정착 가이드만.

### 오래 쓰는 법 (적용 후 정착)

🎯 내 업무에 적용하는 법

  1. **제품 스펙 1장 → 상세페이지 초안** : 성분·임상·FAQ 만 정리해 두면 product_id 하나로 24섹션 초안이 나온다. 외주 전 1차 초안을 사내에서 끝낸다.
  2. **규제 치환 사전을 내 업종으로 교체** : brand-voice.md 의 금기어 표를 식품·건기식·의료기기 등 본인 업종 규제로 바꾸면 같은 게이트가 그대로 작동한다.
  3. **페인 섹션 = VoC 인용 원칙 고정** : 4-4 VoC 산출물을 입력으로 연결하면 "고객이 실제로 쓴 문장"이 카피가 된다. 추측 카피를 줄인다.
  4. **CTA 12종 → A/B 소재 풀** : 생성된 CTA 중 2개를 골라 다변량 테스트 소재로 곧장 쓴다 (6-6 A/B 분석으로 연결).

📊 산출물
  · Notion : 상세페이지 24섹션 + 규제 치환 로그 (검출어·대체어·근거 조항 표 포함) — URL

✅ 체크리스트
  [ ] 입력이 product_id 하나로 충분한 이유 (선행 산출물 자동 로드)
  [ ] 페인 섹션 = VoC 고객 언어 원칙
  [ ] BLOCKED 와 자동 치환의 차이 (치환 불가 표현은 차단)
