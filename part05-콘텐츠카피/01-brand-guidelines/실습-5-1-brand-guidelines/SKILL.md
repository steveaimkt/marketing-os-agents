# Part 5 / 5-1 · 브랜드 가이드라인 에이전트 실습 : 소개 → 실행 → 활용

> 흩어진 브랜드 자산 (사이트·블로그·SNS) 을 분석해 **모든 카피의 기준점이 될 Brand Voice 페이지**를 만든다.
> 핵심 학습 목표 : ① 톤 추출 ② 금기어 체계 (화장품법) ③ "한 번 만들고 모두가 참조" 하는 허브 설계.
> ⛔ 자동 실행 금지. 단계마다 사용자 답을 받은 뒤에만 다음으로.

## 진행 흐름 (3 단계)

```
1단계  소개   하는 일 + Before/After + 연결 소스 + 핵심 포인트     ⏸
2단계  실행   사전 점검 → 작동 구조 → 데이터 준비 → 실제 실행     ⏸
3단계  활용   내 업무 적용법 + 산출물 + 다음 클립                  ⏸
```

---

## 1단계 · 소개

```
🎨 브랜드 가이드라인 에이전트 — 무엇을 하나?

흩어진 브랜드 자산 (메인·소개·제품·블로그·인스타) 을 분석해
"톤·어휘·금기어·페르소나·메시징 필러" 5요소를 추출하고,
이것을 Notion "Brand Voice" 페이지 한 장으로 외부화한다.
핵심 명제 : "톤이 흔들리는 이유는 기준이 머릿속에만 있어서다. 문서로 외부화하라."

⏱ Before vs After
  · Before : 카피 쓸 때마다 톤 고민 + 외주·팀원마다 다른 목소리 + 금기어 사고 위험
  · After  : Brand Voice 페이지 1개 → 모든 에이전트·사람이 같은 기준 참조 = 톤 일관성 + 규제 안전

🔌 연결 소스 (실제 의존만)
  · Notion 통합 (Claude.ai)  — Brand Voice 페이지 생성 (필수)
  · Firecrawl MCP            — 브랜드 자산 스크랩 (실데이터 모드에서만 사용)

🎯 핵심 포인트
  ① 산출물이 "보고서"가 아니라 "다른 에이전트의 입력" : 허브 설계
  ② 금기어가 단순 목록이 아니라 분류 + 대체 표현 쌍 : 차단이 아닌 치환 가능 설계
  ③ 페르소나가 채널·시간대까지 포함 : 카피 채점 (5-5) 의 타겟 적합 축 기준이 됨
```

📐 **스킬? 에이전트?** → 도구(Notion·Firecrawl) + Notion "Brand Voice" 페이지 생성(산출)이 있으니 🤖 **에이전트**(브랜드 자산 추출기). 만든 산출물은 다시 brand-voice **스킬**이 불러 쓴다(허브). 5요소 골격은 3-1과 동일.

⏸ 확인되면 2단계로.

## 2단계 · 실행

### 2-0 사전 점검

```bash
claude mcp list 2>&1 | grep -i "firecrawl"
grep -q "FIRECRAWL_API_KEY" .env 2>/dev/null && echo "key ✅"
```

```
🔧 5-1 의존 항목 점검 결과

  | # | 항목 | 용도 | 상태 | 해결 시간 |
  |---|---|---|---|---|
  | ① | Notion 통합 (Claude.ai) | Brand Voice 페이지 생성       | {✅ / ❌} | 1분 |
  | ② | Firecrawl MCP           | 브랜드 자산 스크랩 (실데이터)  | {✅ / ❌} | 2분 |

  ❌ 항목만 즉시 해결 (4-1 실습과 동일 절차).
  Firecrawl 잔여 크레딧 확인 필수 : 실데이터 모드 30 URL = 30 크레딧 차감.
  샘플 모드는 Firecrawl 불필요 (건너뛰기 허용).
```

### 2-1 작동 구조

```bash
open "agents/part5-copy/brand-guidelines.md"
```

```
🔄 에이전트 워크플로
  [브랜드 자산 30~50개 : 메인·소개·제품·블로그·인스타]
    → Firecrawl 스크랩
    → 분석 : 자주 쓰는 표현 / 문장 길이 / 호칭 / 반복 소재
    → 산출 5요소 : 톤 형용사 3 + 핵심 어휘 20 + 금기어 (화장품법 포함) + 페르소나 2 + 메시징 필러
    → Notion "Brand Voice" 페이지 생성

퀴즈 : "금기어 표에 '대체 표현' 컬럼이 있는 이유는?"
      (답: 5-3 상세페이지에서 금기어를 자동 치환하는 데 사용)
```

### 2-2 데이터 준비

```
📊 데이터 모드:

  [A] 샘플 데이터 모드 ⭐ 추천 · 촬영 기본
      사전 작성된 결로아 Brand Voice 전문 (sample-data/brand-voice.md)을
      Notion 에 시딩. 크레딧 0 소모 · 5분 완료. 분석 과정은 요약 시연.
      구성 : 톤 3 (친근·신뢰·과학) / 어휘 20 / 금기어 표 (화장품법·과장·비교·의학 4분류)
            / 페르소나 2 (지은·수민) / 메시징 필러 4.

  [B] 실데이터 모드
      본인 브랜드 자산 URL 10~50개를 Firecrawl 로 스크랩 후 실분석.
      sample-data/brand-asset-urls.md 양식대로 수집 (최소 10개) → 목록 확정 + 크레딧 견적:
      📋 {N} URL × 1 크레딧 = {N} / 잔여 {M} (월 500). 이대로 진행 OK?
```

> 픽스처 보존 : `sample-data/brand-voice.md` · `sample-data/brand-asset-urls.md` 는 절대 수정하지 않는다. 읽기 전용 시딩 원본.

### 2-3 실제 실행

```
▶ 에이전트 워크플로 단계별 실행:

  [A] 샘플 모드:
  1. brand-voice.md 로드 → 5요소 파싱
  2. "이 5요소가 어떻게 추출되는가" 분석 로직 요약 시연
     (예: 리뷰 빈출어 "순하다·결" → 핵심 어휘 / 화장품법 고시 → 금기어)
  3. Notion "Brand Voice" 페이지 생성 (5요소 섹션 구조)

  [B] 실데이터 모드:
  1. Firecrawl 스크랩 (URL 별 진행 표시 + 크레딧 차감)
  2. 표현·톤 분석 → 5요소 초안
  3. 사용자 검토 : 톤 형용사·금기어 수정 기회        ⏸ 확정 받기
  4. Notion 페이지 생성
```

⏸ Notion 페이지 URL 확인 후 3단계로.

## 3단계 · 활용 — 지금, 내 업무로

### 적용 게이트 (능동 질문 ⏸)

```
🚀 그럼, 지금부터 내 업무에 적용해볼까요?

   ① 어떤 브랜드/업종인가요?
   ② 브랜드 자산을 어디서 줄까요?      (홈페이지·상세페이지·SNS URL 몇 개, 또는 텍스트)
   ③ 우리 업종 규제가 있나요?           (화장품·식품·의료 등 / 없으면 일반)
```

→ 답을 받으면 : 그 자산에서 톤·어휘·금기어·페르소나 추출 → "Brand Voice" 페이지 초안.
→ "건너뛰기" 시 아래 정착 가이드만.

### 오래 쓰는 법 (적용 후 정착)

```
🎯 내 업무에 적용하는 법
  1. 본인 브랜드 자산 URL 10~50개로 실데이터 재실행 → 진짜 내 Brand Voice 확보
  2. 페르소나에 본인 고객 인터뷰·리뷰 인용을 직접 추가 → 실재성 강화
  3. 금기어 표를 우리 업종 규제 (식품·의료기기·금융 등) 로 교체 → 산업 맞춤 안전망
  4. 분기마다 페이지를 갱신 → 신제품·캠페인 톤 변화 반영, 모든 에이전트가 자동 추종

📊 산출물
  · Notion "Brand Voice" 페이지 (톤 3 · 어휘 20 · 금기어 표 · 페르소나 2 · 메시징 필러 4)
  · 이 페이지가 Part 5 모든 카피 에이전트의 공통 기준점(허브)
    — 5-2 (카피 10패턴) · 5-3 (상세페이지) · 5-4 (캘린더) · 5-5 (품질 채점) 전부 자동 로드

✅ 체크리스트
   [ ] 5요소를 나열할 수 있음
   [ ] 금기어 + 대체 표현 쌍 구조의 용도 이해
   [ ] 이 페이지를 참조하는 에이전트 4개를 말할 수 있음

🔁 다음 클립
   → "5-2 실습 시작" : 광고 카피 10패턴 A/B
     (이 Brand Voice 페이지를 즉시 참조하는 첫 카피 에이전트)
```
