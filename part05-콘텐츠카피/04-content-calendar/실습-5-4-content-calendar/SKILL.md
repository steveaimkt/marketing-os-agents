# Part 5 / 5-4 · 콘텐츠 캘린더 에이전트 실습

> 캠페인 일정·트렌드·VoC·키워드를 모아 **다음 주 콘텐츠 30건을 채널별로 기획**하는 에이전트를 가동한다.
> 핵심 학습 목표 : ① 4개 입력 소스 통합 ② 채널 믹스 배분 ③ 기획 (5-4) → 발행 (3-2) 순환.
> ⛔ 자동 실행 금지. 단계마다 사용자 답을 받은 뒤에만 다음으로.

## 진행 흐름 (3 단계)

```
1) 소개   하는 일 + Before/After + 연결 소스        ⏸
2) 실행   작동 구조 + 데이터 준비 + 실제 실행        ⏸
3) 활용   내 업무 적용법 + 산출물 + 다음 클립        ⏸
```

---

## 1단계 · 소개

```
🗓 콘텐츠 캘린더 에이전트 — 이번 주 뭐 올리지?를 없앤다.

하는 일 : 캠페인 일정·트렌드·VoC·SEO 키워드 4개를 모아
          다음 주 콘텐츠 30건을 날짜·채널별로 기획 → Notion Draft 30행으로 적재.

⏱ Before vs After
  · Before : 매주 금요일 기획 회의 2시간 + 글감 고갈 스트레스
  · After  : 금요일 17:00 자동 가동 → 월요일에 Draft 30건 대기 = 검토 15분
```

🔌 연결 소스 (실제 의존만)

```
  | # | 소스 | 용도 | 비고 |
  |---|---|---|---|
  | ① | Notion 통합              | 캘린더 DB 30행 적재  | 산출 저장소 |
  | ② | Google Calendar 통합     | 캠페인 일정 읽기      | OAuth 1클릭 (claude.ai 통합) |
  | ③ | 캠페인 일정 (Calendar)   | 기획의 축            | campaign-calendar.json 4건 시딩 |
  | ④ | SEO TOP 20 (4-5 산출물)  | 글감 소스            | seo-top20.json (✅ sample-data) |
  | ⑤ | VoC TOP 5 (4-4 산출물)   | 글감 소스            | voc-top5.json (✅ sample-data) |
  | ⑥ | 주간 트렌드 (4-2 산출물)  | 글감 소스            | 3-1/4-2 데이터 재사용 또는 픽스처 |
  | ⑦ | Discord webhook          | 기획 요약 발송       | webhook-config.json |
```

💡 핵심 포인트

```
  · 입력이 4개라 "주관"이 아니라 "근거"로 글감이 정해진다.
  · 채널 믹스(IG 7·FB 3·X 14·LinkedIn 2·Blog 1·TikTok 3 = 30)는 상수 정책.
  · 산출 상태가 Draft → 바로 발행이 아니라 3-2 승인 게이트를 다시 거친다 (이중 안전망).
```

📐 **스킬? 에이전트?** → 도구(Notion·Calendar) + 트리거(금 17:00) + 산출(Notion Draft 30행)이 있으니 🤖 **에이전트**(여러 산출물을 모으는 오케스트레이션형). 만드는 법은 3-1의 **에이전트 5요소** 골격 그대로.

⏸ 게이트 확인 : "실행 단계로 갈까요?" 답을 받고 2단계로.

## 2단계 · 실행

### 2-0 · 사전 점검

```bash
claude mcp list 2>&1
test -f "discord-bot/webhook-config.json" && echo "webhook ✅"
```

Google Calendar 는 claude.ai 통합 : `mcp__claude_ai_Google_Calendar__list_calendars` 가벼운 호출로 확인.
❌ 항목은 픽스처로 즉시 시딩 — 캠페인 일정 4건을 Google Calendar 에 생성
(`campaign-calendar.json` · 날짜는 오늘 기준 +3d/+7d/+14d/+21d 환산).

```
📊 데이터 모드:
  [A] 샘플 데이터 모드 ⭐ 추천 · 촬영 기본
      캠페인 4건 (챌린지 시즌2 · 1+1 기획전 · 여름 캠페인 · 패드 티저) +
      픽스처 3종 (트렌드·VoC·키워드)으로 기획 근거가 풍부한 상태에서 시연.
  [B] 실데이터 모드
      본인 Google Calendar 실제 일정 + 선행 실습의 실데이터 사용.
```

⏸ A/B 답 받고 2-1 로.

### 2-1 · 작동 구조 (에이전트 정의 열람)

```bash
open "agents/part5-copy/content-calendar.md"
```

```
📄 이번 클립에서 새로 볼 부분 :
  ① trigger : schedule "매주 금 17:00" — 사람이 쉬는 동안 기획이 도는 설계
  ② 채널 믹스가 상수로 선언됨 — 주관이 아니라 정책 (수정하려면 정의 파일을 고친다)
  ③ 산출 상태가 Draft — 바로 발행이 아니라 3-2 승인 게이트를 다시 거침 (이중 안전망)

  [입력 4소스] Google Calendar(캠페인) · 트렌드(4-2) · VoC(4-4) · SEO(4-5)
    → 채널 믹스 배분 (IG 7 · FB 3 · X 14 · LinkedIn 2 · Blog 1 · TikTok 3 = 30)
    → 각 건 : 날짜 · 채널 · 포맷(캐러셀/릴/트윗/블로그) · 제목 · 키워드 · 예상 분량
    → Notion "콘텐츠 캘린더" 30행 적재 (status=Draft) → Discord 주간 기획 요약
```

퀴즈: "30건이 자동 발행되지 않는 이유는?" (답: Draft 상태 → 3-2 승인 게이트 통과 필요)

⏸ 답 받고 2-2 로.

### 2-2 · 데이터 준비 (선행 산출물 픽스처)

```
  1. Google Calendar 캠페인 4건 시딩 결과 (다음 주에 걸리는 일정 강조)
       · 챌린지 시즌2(+3d) · 1+1 기획전(+7d) · 여름 캠페인(+14d) · 패드 티저(+21d)
  2. seo-top20.json 로드 — 글감 후보 (1위 "모공 토너 추천" 22,100 / 8위 "닦토 하는 법" shorts)
  3. voc-top5.json 로드 — 고객 관심사 (1위 "용기·펌프 불편" → 패드 티저 연결)
  4. 주간 트렌드 — 3-1/4-2 실습 데이터 재사용 또는 픽스처 (모드 [A]/[B] 보존)
  5. Notion "콘텐츠 캘린더" DB 확인 (3-2 실습에서 생성했으면 재사용)
```

⏸ OK 받고 2-3 으로.

### 2-3 · 실제 실행 (시연 하이라이트)

```
▶ 에이전트 워크플로 단계별 실행:
  1. 입력 4소스 로드 요약
  2. 다음 주 7일 × 채널 믹스 배분 계획     → 날짜×채널 매트릭스 표
  3. 30건 기획 생성 (하이라이트)           → 대표 6건의 기획 근거 풀이:
       · 월 IG 캐러셀 "21일 챌린지 인증 가이드" ← 캠페인(챌린지 시즌2) + SEO 6위
       · 화 Blog "모공 토너 고르는 기준 5가지" ← SEO 1위 (검색량 22,100)
       · 수 X "펌프 불편하셨죠" 예고 ← VoC 1위 → 패드 티저 캠페인 연결
       · 목 IG 릴 "30초 닦토 루틴" ← SEO 8위 shorts 추천
       · 금 TikTok "여름 유분 케어" ← 시즌 캠페인
       · 일 LinkedIn "D2C 챌린지 마케팅 회고" ← 브랜드 필러
  4. Notion 30행 적재 (status=Draft)
  5. Discord 주간 기획 요약 embed (채널 분포 + TOP 3 글감 근거)
```

⏸ 결과 확인 후 3단계로.

## 3단계 · 활용 — 지금, 내 업무로

### 적용 게이트 (능동 질문 ⏸)

```
🚀 그럼, 지금부터 내 업무에 적용해볼까요?

   ① 어떤 브랜드/업종인가요?
   ② 운영 채널과 주간 발행 빈도는?      (예: 블로그 3 · IG 2)
   ③ 기획에 반영할 입력이 있나요?        (트렌드·SEO·VoC / 없으면 샘플)
```

→ 답을 받으면 : 다음 주 콘텐츠 캘린더 Draft 표 + 채널 분포.
→ "건너뛰기" 시 아래 정착 가이드만.

### 오래 쓰는 법 (적용 후 정착)

🎯 내 업무 적용법

```
  1. 채널 믹스를 본인 운영 채널·발행 빈도에 맞게 정의 파일에서 수정한다
     (예: 블로그 중심이면 Blog 3 · IG 2 로 재배분).
  2. 입력 소스를 본인 자산으로 교체한다 — 사내 캠페인 캘린더 + 자사몰 검색어 + CS 인입.
  3. trigger 를 "매주 금 17:00" 로 걸어 월요일 검토 15분 루틴으로 운영한다.
  4. Draft → 승인 → 발행의 이중 안전망을 유지해 자동 발행 사고를 차단한다.
```

📊 산출물

```
  · Notion  : 콘텐츠 캘린더 30행 (Draft · 기획 근거 포함)
  · Discord : 주간 기획 요약 (채널 분포 + TOP 3 글감 근거)
```
