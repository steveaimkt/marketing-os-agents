# Part 5 / 5-5 · 품질 채점 6축 실습 : 소개부터 3종 판정까지

> 카피를 **6축 (명확성·타겟·차별성·CTA·브랜드 일관성·규제 안전성) 으로 점수화**하는 에이전트를 가동한다.
> 핵심 학습 목표 : ① 6축 척도 ② 소프트 게이트 (수정 권고) vs 하드 게이트 (차단) ③ 채점 결과의 재사용.
> ⛔ 자동 실행 금지. 각 단계가 끝날 때마다 사용자 답을 받은 뒤에만 다음으로.

## 진행 흐름 (3 단계)

```
1단계  소개   하는 일 + Before/After + 연결 소스          ⏸ 답 대기
2단계  실행   작동 구조 → 데이터 준비 → 실제 실행 (3종 판정) ⏸ 답 대기
3단계  활용   내 업무 적용법 + 산출물 + 다음 클립           ⏸ 선택
```

## 1단계 · 소개

```
⚖️ 품질 채점 6축 실습을 시작합니다.

본 흐름의 핵심 : "좋은 카피냐는 취향 논쟁을 6개 축의 점수로 바꾼다"

⏱ Before vs After
  · Before : 카피 검토 = 상급자 감각 의존 + 규제 검토는 별도 (놓치면 사고)
  · After  : 6축 점수 + 수정 제안 + 규제 자동 차단 = 30초, 기준이 기록으로 남음

🔌 연결 소스 (이 실습이 실제로 쓰는 것만)
  ┌─ Skill : quality-review-6axis   → 6축 채점 로직 (척도·판정 규칙)
  ├─ Notion "Brand Voice" 페이지     → 브랜드 일관성·규제 축의 기준점
  └─ Discord webhook                → 6축 점수 카드 발송

🔑 핵심 포인트
  · 소프트 게이트 (수정 권고) 와 하드 게이트 (차단) 의 분리 = 의사결정 권한의 분리
  · 채점기가 5-2 (10패턴 선별) · 5-3 (규제 게이트) · 3-1 (발송 전 검증) 에
    이미 들어가 있었다 → Part 5 의 마지막 퍼즐이자 전체 품질의 바닥

👉 2단계로 넘어가 작동 구조부터 볼까요?
```

📐 **스킬? 에이전트? — 둘 다 나온다**

```
· 6축 '채점 로직(척도·판정 규칙)' = 순수 변환 → 🧩 스킬 quality-review-6axis
· 그걸 불러 Discord 점수 카드 발송 = 산출 있음 → 🤖 에이전트가 감쌈
⇒ rubric 은 스킬, 피드백 전달은 에이전트 — 둘의 분업을 직접 본다
```

### 2-1 작동 구조 (학습 핵심)

```bash
open "agents/part5-copy/quality-reviewer-6axis.md"
```

```
🔄 에이전트 워크플로
  [카피 : 헤드라인 + 본문 + CTA + 맥락 (채널·타겟·목표)]
    → Brand Voice 자동 로드
    → 6축 채점 (각 0~5):
       명확성 / 타겟 적합 / 차별성 / CTA 강도 / 브랜드 일관성 / 규제 안전성
    → 판정 :
       · 평균 3.5 이상 + 규제 3 이상 = 통과
       · 평균 3.5 미만 = REVISION_RECOMMENDED + 수정안 1건 자동 생성 (소프트 게이트)
       · 규제 2 이하 = BLOCKED · 발행 불가 (하드 게이트 · 사람도 못 넘김)
    → Discord 점수 카드

📄 이번 클립에서 새로 볼 부분 :
  ① 6축 각각의 채점 질문이 명문화 ("헤드라인에 핵심 메시지가 있는가" 등)
     → LLM 채점의 신뢰도는 척도의 구체성에서 나온다
  ② 소프트 게이트 vs 하드 게이트 : 품질은 권고, 규제는 차단 (의사결정 권한 분리)
  ③ 수정안 자동 생성 : 점수만 주지 않고 다음 행동까지 제공

🧩 퀴즈 : "평균 4.5 인데 규제 축만 1점이면 판정은?" (답: BLOCKED · 평균과 무관)
```

### 2-2 데이터 준비

```
📊 데이터 모드:

  [A] 샘플 데이터 모드 ⭐ 추천 · 촬영 기본
      sample-data/review-targets.json 의 채점 대상 3건을 그대로 사용 (수정 금지).
      통과 예상 (COPY-A) · 수정 권고 예상 (COPY-B) · 차단 예상 (COPY-C).
      3가지 판정이 모두 나오도록 설계된 세트.

  [B] 실데이터 모드
      본인이 쓴 카피 (헤드라인 + 본문 + CTA) 를 직접 입력해 채점.
```

`sample-data/review-targets.json` 로드, 3건을 나란히 표시 (예상 판정은 가리고):

```
📥 채점 대상 3건 (맥락 : Meta · 20~30 여성 모공·유분 고민 · 전환)

  COPY-A : "모공 고민, 21일이면 충분해요" + PHA·임상 수치 본문 + 챌린지 CTA
  COPY-B : "신학기 세일 30%" + 토너 1+1 본문 + 지금 구매 CTA
  COPY-C : "모공 축소 + 주름 개선 토너" + 기적·재생·치료 본문

  어떤 카피가 통과할지 예상해 보세요 (시연 인터랙션 포인트).
```

⏸ 모드 선택 + 예상 답 받고 2-3 으로.

### 2-3 실제 실행

```
▶ 3건 순차 채점 (각 건마다 결과 확인):

  COPY-A → 6축 카드 : 평균 4.2 · 전축 3.5+ → ✅ 통과
            (임상 수치 인용이 차별성·명확성을 끌어올린 근거 풀이)

  COPY-B → 6축 카드 : 평균 2.8 (명확성 2 · 타겟 2 · 차별성 1)
            → ⚠️ REVISION_RECOMMENDED + 수정안 자동 생성:
            "신학기 모공 케어, 포어 토너 1+1" 식으로 제품·타겟 명시 (재채점 시 상승 확인)

  COPY-C → 규제 축 0 (위반 6건 : 모공 축소·주름 개선·재생·치료·기적의·단 하루 만에)
            → ⛔ BLOCKED : 발행 절대 불가 + 화장품법 조항 근거 + 2-2 예상과 대조

  → Discord 점수 카드 3건 발송
```
