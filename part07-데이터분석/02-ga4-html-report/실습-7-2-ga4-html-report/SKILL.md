# Part 7 / 7-2 · GA4 HTML 리포트 에이전트 실습 : 소개 → 실행 → 활용

> 7-1이 만든 **시트 5탭을 받아 KPI·차트·표·액션이 담긴 단일 HTML 리포트 1장**으로 시각화한다.
> 핵심 학습 목표 : ① KPI 4종 읽기 ② 차트 3종의 의미 ③ Part 6과 같은 html-report-template 스킬 재사용(DRY).
> ⛔ 자동 실행 금지. 단계마다 사용자 답을 받은 뒤에만 다음으로.

## 진행 흐름 (3 단계)

```
1단계  소개  — 하는 일 + Before/After + 연결 소스 + 핵심 포인트       ⏸ 답 대기
2단계  실행  — 작동 구조 + 데이터 준비 + 실제 실행 (시트 → HTML)      ⏸ 답 대기
3단계  활용  — 내 업무 적용법 + 산출물 + 다음 클립                     ⏸ 선택
```

## 1단계 · 소개

```
📈 GA4 HTML 리포트 에이전트 실습을 시작합니다.

🎯 하는 일
  "숫자 시트를 열어보라고 공유하지 않는다. 폰에서도 읽히는 리포트 1장을 보낸다."
  7-1이 정리한 시트 5탭을 받아 → KPI 4종 + 차트 3종 + 페이지 TOP 표 + 권장 액션을
  담은 단일 HTML 리포트로 만든다. (외부 의존 없는 단일 파일 = 오프라인·폰에서도 열림)

⏱ Before vs After
  · Before : 시트 열기 → 그래프 수동 작성 → 캡처 → 문서 편집 = 약 30분/회
  · After  : 7-1 완료 후 자동 (또는 명령 1줄) → HTML 리포트 = 1분
```

🔌 연결 소스 (트리거 시 점검 · 실제 의존만)

```bash
claude mcp list 2>&1 | grep -i "google-sheets"
test -f "skills/html-report-template/SKILL.md" && echo "리포트 템플릿 ✅"
```

```
  | # | 항목 | 용도 | 상태 |
  |---|---|---|---|
  | ① | Google Sheets MCP        | 7-1 시트 5탭 로드      | {✅ / ❌} |
  | ② | html-report-template 스킬 | 리포트 디자인 (공유)   | {✅ / ❌} |

  ❌ html-report-template 미확인 시:
  `ls skills/html-report-template/SKILL.md` 로 존재 확인
   → Part 6(6-1~6-4)과 동일한 스킬 · 새로 설치할 것 없음.
   → 샘플 모드는 ① 없이도 진행 가능 (내장 요약 JSON 사용).
```

📐 **스킬? 에이전트?** → 도구(Sheets) + 산출(HTML) + 다음 단계 전달이 있으니 🤖 **에이전트**. 리포트 디자인은 Part 6과 **같은 html-report-template 스킬**을 공유한다(광고·GA4 리포트가 한 디자인 = DRY).

💡 핵심 포인트
  · KPI 4종 = 총 세션 · 신규/재방문 비율 · 전환수 · 매출
  · 차트 3종 = 일별 세션·전환 선차트 / 채널 세션 도넛 / 장치별 막대
  · 🎨 같은 템플릿 재사용 — 광고 리포트(Part 6)와 시각 언어가 통일됨

⏸ 여기까지 이해되셨으면 "2단계" 라고 답해 주세요.

## 2단계 · 실행

### 2-0 · 사전 점검
1단계의 연결 소스 표에서 ❌ 인 항목을 먼저 해결한다.
샘플 모드면 Sheets MCP ❌ 라도 내장 요약 JSON 으로 진행 가능. (html-report-template 은 필수)

### 2-1 · 작동 구조
```
  [7-1 시트 5탭 (또는 요약 JSON)]
    → 로드 : 채널·페이지·전환경로·신규재방문·장치 + 일별 추세
    → KPI 4종 산출 : 총 세션 · 신규/재방문 비율 · 전환수 · 매출 (WoW 화살표)
    → html-report-template 스킬 호출 (report_type: "ga4")
        · KPI 카드 4 + 차트 3종(인라인 · 외부 의존 없음)
            ① 일별 세션·전환 선차트 (2축)
            ② 채널 세션 점유 도넛
            ③ 장치별 세션·전환율 막대
        · 페이지 TOP 20 표 + 권장 액션 5
    → outputs/{날짜}/ga4-weekly-{날짜}.html 저장
    → 경로를 다음 에이전트(7-3)에 전달
```
📄 에이전트 정의에서 새로 볼 부분 (`agents/part7-ga4/ga4-html-report.md`):
  ① report_type "ga4" — 같은 템플릿을 광고 아닌 GA4 용으로 호출
  ② 차트 3종이 GA4 특성(추세·채널·장치)에 맞게 지정됨
  ③ 단일 파일 HTML — Chart 인라인·외부 의존 없음 → 폰·오프라인에서 그대로 열림

퀴즈: "이 리포트를 왜 단일 HTML 파일로 만들까?"
(답: 링크·CDN 없이 첨부 하나로 전달 → 7-3에서 디스코드·노션에 그대로 실림)

### 2-2 · 데이터 준비

```
📊 데이터 모드:

  [A] 샘플 데이터 모드 ⭐ 추천 · 촬영 기본
      sample-data/ga4-sheet-summary.json 로드 (7-1이 만든 시트 요약과 동일 수치).
      결로아 가상 속성 · W26(2026-06-22~28) · 5탭 + 일별 추세.
      실속성 수치가 영상에 노출되지 않고, KPI·차트·표가 선명하게 재현됨.
      ※ 픽스처는 절대 수정하지 않는다 (재현성 보존).

  [B] 실데이터 모드
      7-1을 본인 GA4 속성으로 먼저 실행 → 그 시트 5탭을 입력으로 사용 (읽기 전용).
      ⚠️ 촬영 시 속성 지표가 리포트에 노출됨을 확인하세요.
```

⏸ A/B 답 받고 2-3 으로.

### 2-3 · 실제 실행
```
▶ 에이전트 워크플로 단계별 실행:

  1. 시트 5탭(또는 요약 JSON) 로드
  2. KPI 4종 산출                          → (시연 하이라이트)
       · 총 세션 37,800 (▲3.8%) · 신규/재방문 71 / 29
       · 전환수 1,168 (▲8.6%) · 매출 38,050,000원 (▲10.4%)
  3. html-report-template 호출 → HTML 생성  → outputs/{날짜}/ga4-weekly-{날짜}.html
       → 브라우저로 열어 확인
       · 차트 3종 (인라인 · 외부 의존 없음 → 단일 파일·오프라인 OK)
           ① 일별 세션·전환 선차트 : 06-22~28 · 금요일 5,900 피크
           ② 채널 세션 도넛 : Organic 38% · Paid 26% · Direct 16% · Social 14% · Referral 6%
           ③ 장치별 막대 : 모바일 세션 68%지만 전환율 2.46% (데스크톱 4.58%) ← 갭 시각화
       · 페이지 TOP 20 표 : /products/pore-toner-v2 전환율 5.5% 최상단
       · 권장 액션 5 : 유료 증액·모바일 결제 A/B·리텐션·LP 집중·콘텐츠 CTA
  4. HTML 경로 확보                          → 7-3(ga4-notion-publisher)의 입력으로 전달
```

⏸ 리포트 화면 확인 후 "3단계" 라고 답해 주세요.
