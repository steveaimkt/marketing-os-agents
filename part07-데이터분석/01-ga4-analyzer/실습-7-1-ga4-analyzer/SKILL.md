# Part 7 / 7-1 · GA4 분석 에이전트 실습 : 소개 → 실행 → 활용

> GA4 속성에서 **채널·페이지·전환·신규재방문·장치 5개 데이터셋을 읽기 전용으로 추출해 Google Sheets 5탭**으로 정리한다.
> 핵심 학습 목표 : ① 5개 데이터셋의 의미 ② 세션→전환→매출 정합 읽기 ③ 다음 에이전트(7-2)로 넘어가는 파이프라인 첫 칸.
> ⛔ 자동 실행 금지. 단계마다 사용자 답을 받은 뒤에만 다음으로.

## 진행 흐름 (3 단계)

```
1단계  소개  — 하는 일 + Before/After + 연결 소스 + 핵심 포인트       ⏸ 답 대기
2단계  실행  — 작동 구조 + 데이터 준비 + 실제 실행 (5탭 → 시트)       ⏸ 답 대기
3단계  활용  — 내 업무 적용법 + 산출물 + 다음 클립                     ⏸ 선택
```

---

## 1단계 · 소개

```
📊 GA4 분석 에이전트 실습을 시작합니다.

🎯 하는 일
  "GA4 화면 이곳저곳 클릭하지 않는다. 매주 같은 5개 표가 시트에 채워져 있다."
  GA4 속성을 읽기 전용으로 조회해 → 채널·페이지·전환경로·신규재방문·장치 5개
  데이터셋을 뽑고 → Google Sheets 5탭 + Summary 탭으로 정리한다. (7-2 리포트의 입력)

⏱ Before vs After
  · Before : GA4 접속 → 보고서 5개 이동 → 기간·측정기준 설정 → 시트 복붙 = 약 35분/회
  · After  : 명령 1줄 (또는 매주 월 자동) → 시트 5탭 자동 정리 = 1분
```

🔌 연결 소스 (트리거 시 점검 · 실제 의존만)

```bash
claude mcp list 2>&1 | grep -i "ga4"
claude mcp list 2>&1 | grep -i "google-sheets"
```

```
  | # | 항목 | 용도 | 상태 |
  |---|---|---|---|
  | ① | GA4 MCP                   | 속성 데이터 조회 (읽기 전용) | {✅ / ❌} |
  | ② | Google Sheets MCP        | 5탭 + Summary 정리          | {✅ / ❌} |
  | ③ | Discord webhook (선택)    | 주간 요약 알림              | {✅ / ❌} |

  ❌ GA4 MCP 미등록 시:
  `claude mcp add ga4 ...` → 첫 호출 시 OAuth (✋ 사용자) → 속성 목록 조회로 검증
   → ⚠️ 신규 등록 시 세션 재시작 1회. 샘플 모드는 ①·② 없이도 진행 가능 (건너뛰기 허용).
```

📐 **스킬? 에이전트?** → 도구(GA4·Sheets) + 산출(시트 5탭) + 다음 단계 전달이 있으니 🤖 **에이전트**. 이 에이전트는 파이프라인 **첫 칸** — 7-2(HTML)·7-3(발송)의 원재료를 만든다.

💡 핵심 포인트
  · 5개 데이터셋 = 채널 / 페이지 TOP / 전환 경로 / 신규 vs 재방문 / 장치별
  · 세션 × 전환율 ≈ 전환, 채널·장치·신규재방문 합계 = 총계 — 정합이 맞아야 신뢰
  · 🛡 읽기 전용 : 이 에이전트는 GA4 설정·이벤트를 절대 바꾸지 않는다

⏸ 여기까지 이해되셨으면 "2단계" 라고 답해 주세요.

## 2단계 · 실행

### 2-0 · 사전 점검
1단계의 연결 소스 표에서 ❌ 인 항목을 먼저 해결한다.
샘플 모드면 GA4·Sheets MCP ❌ 라도 그대로 진행 가능.

### 2-1 · 작동 구조
```
  [속성 + 기간 (기본 최근 7일) + 전주 비교]
    → GA4 조회 (run_report 5회, 서로 독립 → 병렬 팬아웃)
        ① 채널 성과   : 채널 × 세션·전환·전환율·매출
        ② 페이지 TOP  : 랜딩 × 세션·전환·이탈률
        ③ 전환 경로   : 소스/매체 → 랜딩 → 전환
        ④ 신규 vs 재방문
        ⑤ 장치별      : Desktop / Mobile / Tablet
    → 정합 검증 (합계 = 총계, 세션×전환율 ≈ 전환)
    → Google Sheets 5탭 저장 + Summary 탭 핵심 수치 요약
    → 결과 시트 URL 을 다음 에이전트(7-2)에 전달
```
📄 에이전트 정의에서 새로 볼 부분 (`agents/part7-ga4/ga4-analyzer.md`):
  ① run_report 5회 = 데이터셋 5개 — 각 탭이 무엇을 측정하는지 명시
  ② Summary 탭 — 5탭의 핵심 수치를 한 곳에 모아 다음 에이전트가 바로 읽음
  ③ 병렬 팬아웃 — 독립 조회는 동시에 (공통 규칙: 수집=경량 모델)

퀴즈: "이 에이전트에게 'GA4에서 전환 이벤트 하나 추가해줘'라고 하면?"
(답: 거부. 읽기 전용 원칙 + 설정 변경은 사용자 영역)

### 2-2 · 데이터 준비

```
📊 데이터 모드:

  [A] 샘플 데이터 모드 ⭐ 추천 · 촬영 기본
      sample-data/ga4-weekly-raw.json 로드 (결로아 가상 GA4 속성).
      5개 데이터셋 + 일별 추세 · 최근 7일(2026-06-22~28) + 전주 비교.
      실속성 수치가 영상에 노출되지 않고, 유료채널 상승·모바일 저조·재방문 고전환
      신호가 보장되어 시연이 선명함.
      ※ 픽스처는 절대 수정하지 않는다 (재현성 보존).

  [B] 실속성 모드
      본인 GA4 속성 라이브 조회 (읽기 전용 · 설정 변경 절대 없음).
      속성 목록 조회 → 대상 속성 선택 ⏸ → 최근 7일 데이터 존재 확인.
      ⚠️ 촬영 시 속성 지표가 화면에 노출됨을 확인하세요.
```

⏸ A/B 답 받고 2-3 으로.

### 2-3 · 실제 실행
```
▶ 에이전트 워크플로 단계별 실행:

  1. 데이터 로드 (5개 데이터셋 + 일별, 7일 + 전주)
  2. 총계 정합 확인                       → (시연 하이라이트)
       · 세션 37,800 (+3.8%) · 전환 1,168 (+8.6%) · 매출 38,050,000원 (+10.4%)
       · 채널 합 = 장치 합 = 신규+재방문 합 = 37,800 (정합 OK)
  3. 5탭 채우기                            → Google Sheets "GA4 분석"
       ① 채널     : Paid Search 매출 +25.6%·전환율 3.96%(최고) ← 유료 상승 신호
       ② 페이지   : /products/pore-toner-v2 전환율 5.5% 최고 / 블로그 이탈률 68%
       ③ 전환경로 : google/cpc→챌린지 142 · direct·organic→포어토너 상세
       ④ 신규재방문 : 재방문 전환율 6.02% = 신규 1.90%의 3.2배
       ⑤ 장치     : 모바일 세션 68%인데 전환율 2.46% = 데스크톱 4.58%의 절반 ← 저조 신호
  4. Summary 탭                            → 5탭 핵심 수치 + WoW 요약 1행
  5. 결과 시트 URL 확보                    → 7-2(ga4-html-report)의 입력으로 전달
```

⏸ 시트 5탭 확인 후 "3단계" 라고 답해 주세요.
