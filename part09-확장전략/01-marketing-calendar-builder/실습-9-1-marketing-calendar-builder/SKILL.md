# Part 9 / 9-1 · 마케팅 캘린더 구축 에이전트 실습 : 소개 → 실행 → 활용

> **브랜드 자체 일정을 앵커로 먼저 깔고**, 그 위에 카테고리 계절성·한국 기념일·SEO 키워드를 결합해 **1년 캠페인 50~60개를 설계**한다. 노션에 바로 쓰지 않고 **표(초안)로 먼저 검토**한 뒤 승인되면 노션 캘린더 + Google Calendar 에 동기화한다.
> 핵심 학습 목표 : ① 브랜드 일정 우선 앵커링 ② 12개월 표준 컴포넌트 × 카테고리 결합 ③ SEO 키워드의 시즌 분배 ④ 표-검토 승인 게이트 후 노션(날짜 속성→캘린더 뷰) 동기화.
> ⛔ 자동 실행 금지. 단계마다 사용자 답을 받은 뒤에만 다음으로. 특히 노션 생성 전 표 검토는 필수 게이트.

## 진행 흐름 (3 단계)

```
1단계  소개  — 하는 일 + Before/After + 연결 소스 + 핵심 포인트          ⏸ 답 대기
2단계  실행  — ① 브랜드 일정 먼저 입력 → ② 연간 이벤트 결합
              → ③ 표(초안) 검토(추가·제외) → ④ 승인 후 노션·캘린더 동기화  ⏸ 답 대기
3단계  활용  — 내 업무 적용법 + 산출물 + 다음 클립                        ⏸ 선택
```

## 1단계 · 소개

```
🗓 마케팅 캘린더 구축 에이전트 실습을 시작합니다.

🎯 하는 일
  "내년 뭘 언제 밀지 12월에 몰아서 고민하지 않는다. 1년치가 캘린더에 깔린다."
  카테고리 계절성 + 한국 기념일 + SEO 키워드를 종합해 → 12개월 × 4~6 캠페인 카드
  (기간·채널·페르소나·핵심 메시지·SEO 키워드·예상 예산) 50~60개로 설계하고
  노션 캘린더 DB + Google Calendar 에 한 번에 동기화한다.

⏱ Before vs After
  · Before : 작년 자료 뒤지기 → 기념일 검색 → 엑셀 월별 칸 채우기 = 반나절/년
  · After  : 명령 1줄 → 캠페인 카드 50~60개 + 캘린더 2곳 동기화 = 3분
```

🔌 연결 소스 (트리거 시 점검 · 실제 의존만)

```bash
claude mcp list 2>&1 | grep -i "notion"
claude mcp list 2>&1 | grep -i "calendar"
test -f "agents/part9-expansion/marketing-calendar-builder.md" && echo "에이전트 정의 ✅"
```

```
  | # | 항목 | 용도 | 상태 |
  |---|---|---|---|
  | ① | SEO 키워드 (4-5 산출 / 샘플)   | 시즌 키워드 분배 원천 | {✅ / ❌} |
  | ② | Notion 통합                   | 연간 캘린더 DB        | {✅ / ❌} |
  | ③ | Google Calendar 통합          | 마일스톤 등록         | {✅ / ❌} |
  | ④ | Agent(seo-keyword-research)   | 키워드 실시간 확장(옵션) | {✅ / ❌} |

  ❌ Google Calendar 미연결 시:
  claude.ai 커넥터 설정에서 Google Calendar 승인 (✋ 사용자) → 캘린더 목록 조회로 검증.
   → 샘플 모드는 ③④ 없이도 진행 가능 (노션 DB 까지만, 캘린더는 건너뛰기 허용).
```

📐 **스킬? 에이전트?** → 입력(키워드·기념일) + 도구(Notion·Calendar) + 산출(카드·DB·일정)이 있으니 🤖 **에이전트**. 키워드 확장이 필요하면 seo-keyword-research **에이전트**를 호출한다(9-1 이 상위, 4-5 가 하위 = 재사용). 이 캘린더가 9-2·9-3 의 입력이 된다.

💡 핵심 포인트
  · 🥇 **브랜드 자체 일정을 먼저 앵커로** 깔고, 그 위에 연간 이벤트를 결합 (주객 전도 방지)
  · 12개월 표준 컴포넌트(신학기·가정의달·블프…)는 고정, 카테고리 키워드만 매핑해 조합
  · SEO 키워드는 검색량이 아니라 **시즌 태그**로 배치 (전기톱=가을, 전지가위=봄 처럼)
  · 🛡 브랜드 앵커 충돌 점검 : 출시일과 겹치는 캠페인을 자동 표시 (제안까지만)
  · 📋 **표 검토 게이트** : 노션 생성 전 표(초안)로 먼저 보여주고 추가·제외 승인받은 뒤에만 기록

## 2단계 · 실행

### 2-0 · 사전 점검
1단계의 연결 소스 표에서 ❌ 인 항목을 먼저 해결한다.
샘플 모드면 Google Calendar ❌ 라도 노션 DB 까지 진행 가능 (캘린더 동기화만 건너뜀).

### 2-1 · 작동 구조
```
  ① [브랜드 자체 일정·플랜 — 먼저 입력]  ← 앵커 (출시일·자체 이벤트·프로모션·고정 리듬)
    → 브랜드 일정을 캘린더 앵커로 먼저 배치
  ② [연간 이벤트(기념일·계절성) + 카테고리 + SEO 키워드 — 결합]
    → 12개월 표준 컴포넌트에 카테고리 키워드 매핑 (앵커 위에 얹음)
    → SEO 우선순위 키워드를 시즌별로 분배 (계절성 태그 기준)
    → 브랜드 앵커와 캠페인 충돌·시너지 점검 (겹침 표시)
    → 캠페인 카드 50~60개 생성 (월 4~6개)
        · title / period / channels / persona / core_message / seo_keywords / estimated_budget
  ③ [표(초안)로 먼저 출력 → 사용자 추가·제외·수정 검토]  ⏸ 승인 게이트 (필수)
    → "확정" 받을 때까지 반영·재출력 반복
  ④ [확정 후에만 동기화]
    → Notion "{년도} 마케팅 캘린더" DB 일괄 입력
        · 기간을 날짜(Date) 속성으로 입력 → 캘린더 뷰 생성 (달력 위 기간 막대)
    → Google Calendar "Marketing OS" 에 마일스톤 등록 (선택)
```
📄 에이전트 정의에서 새로 볼 부분 (`agents/part9-expansion/marketing-calendar-builder.md`):
  ① 12개월 표준 컴포넌트 표 — 계절성 골격은 고정, 카테고리만 갈아끼움 (재사용성)
  ② 캠페인 카드 YAML 표준 — 7개 필드 고정 → 9-2·9-3 이 그대로 읽어 씀
  ③ 충돌 점검 단계 — 출시 일정과 캠페인이 겹치면 "제안"으로 표시 (실행 아님)

퀴즈: "6월 신제품 출시가 이미 잡혀 있으면 에이전트는?"

### 2-3 · 실제 실행
```
▶ 에이전트 워크플로 단계별 실행:

  1. ⏸ 브랜드 일정 먼저 질문·입력 (출시일·자체 이벤트·프로모션·고정 리듬) → 캘린더 앵커로 배치
  2. 나머지 입력 로드 (카테고리 + 기념일표 + SEO 키워드)
  3. 12개월 매핑표 작성 (브랜드 앵커 위에 얹음)   → 월×핵심 캠페인 후보 (시연 하이라이트)
       · 3월 신학기 → "신학기 모공 케어 21일 챌린지" (토너 모공·21일 챌린지)
       · 5월 가정의달 → "엄마 피부 선물 세트" (선물·민감 토너)
       · 11월 블프 → "연중 최대 토너 리필 대전" (리필·대용량)
  4. SEO 키워드 시즌 분배                  → 검색량 아닌 시즌 태그로 배치
  5. 브랜드 앵커 충돌·시너지 점검            → 예: 9월 신제품 출시 vs 9월 환절기 캠페인 겹침 표시
  6. 캠페인 카드 50~60개 생성
  7. ⏸ 표(초안)로 먼저 출력 → 사용자 추가·제외·수정 검토 (승인 게이트 · 필수)
       · 월 / 캠페인명 / 기간 / 채널 / 페르소나 / 예산 / 충돌 컬럼의 표로 제시
       · 사용자가 "이 캠페인 빼줘 / 추가해줘 / 예산 바꿔줘" 지시 → 반영 후 재출력
       · "확정" 답을 받을 때까지 반복 (노션 생성은 이 게이트 통과 전 절대 금지)
  8. [확정 후에만] Notion "2027 마케팅 캘린더" DB 일괄 입력 (마케팅 OS 시작 페이지 하위)
       · 기간을 날짜(Date) 속성으로 입력 (date:일정:start / end) → 캘린더 뷰 생성 (달력 위 기간 막대)
       · 충돌 표시 카드는 충돌 체크박스 ON → 필터로 구분
  9. Google Calendar "Marketing OS" 에 캠페인 시작일 마일스톤 등록 (선택)
       · 충돌 표시된 일정은 색상/메모로 구분 → 사용자 조정 대기
```
