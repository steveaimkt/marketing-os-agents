# 프로토타입 브리프 — 예시 프롬프트 3종 + 분해 표

> 9-2 실습은 이 아이디어를 입력받아 **클로드코드가 design-brief.md + 복붙용 프롬프트를 산출**하고,
> 그 프롬프트를 **클로드 디자인(Artifacts)** 에 붙여넣어 프로토타입을 생성한다(핸드오프).
> 복붙 프롬프트 골격은 `paste-prompt-sample.md`, 완성 결과 레퍼런스는 `sample-output/index.html`.

## 예시 프롬프트 3종 (2-2 에서 택 1, 또는 사용자 직접 입력)

### 1) 진단형 미니앱 ⭐ 대표 시연
> "신학기 모공 진단 미니 앱: 3문항 답 → 결로아 토너 추천 페이지"

### 2) 랜딩 프로토타입
> "21일 챌린지 신청 랜딩 : 히어로 + 혜택 3개 + 신청 폼 (이름·이메일)"

### 3) 인터랙티브 계산기
> "토너 리필 절약 계산기 : 월 사용량 입력 → 정가 대비 리필 연간 절약액 표시"

## 4컴포넌트 분해 표 (예시 1 기준)

| 컴포넌트 | 설계 |
|---|---|
| **화면 (screen)** | ① 인트로/진단 (3문항 순차) → ② 결과 추천 (1개 화면 전환) |
| **상태 (state)** | `step` (0~2 문항 인덱스) · `answers[]` · `score` 합산 |
| **입력 (input)** | 문항당 라디오 3택 (0·1·2점 가중) |
| **결과 (result)** | 점수 구간 분기 → 모공(포어 토너 V2) / 수분(워터 토너) / 민감(카밍 토너) |

## 3문항 (예시 1 스크립트)

1. **세안 후 몇 시간 뒤 번들거리나요?** → 바로(모공2) / 3~4시간(1) / 거의 안 남(수분0)
2. **모공이 가장 신경 쓰이는 부위는?** → T존·볼(모공2) / 부분적(1) / 딱히 없음(민감 가중) 
3. **자극·붉어짐이 있나요?** → 자주(민감2) / 가끔(1) / 없음(0)

## 결과 분기 규칙 (예시 1)

| 점수/신호 | 추천 | 카피 |
|---|---|---|
| 모공 신호 우세 | 포어 토너 V2 | "T존까지 조이는 모공 집중 케어" |
| 수분 신호 우세 | 워터 토너 | "속당김 없는 수분 결 채우기" |
| 민감 신호 우세 | 카밍 토너 | "자극 없이 진정부터, 저자극 포뮬러" |

## 9-1 연결 (선택)
9-1 캠페인 카드(예: "신학기 모공 케어 21일 챌린지")의 `core_message`·`persona`·`seo_keywords`
를 히어로 카피/타겟으로 넣으면 캠페인 랜딩 프로토타입이 된다.
