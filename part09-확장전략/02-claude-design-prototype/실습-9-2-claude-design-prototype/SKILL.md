# Part 9 / 9-2 · Claude Design 프로토타입 실습 : 소개 → 실행 → 활용

> 마케터의 아이디어 한 줄을 **클로드코드가 기획·프롬프트로 다듬고 → 클로드 디자인이 클릭 가능한 프로토타입으로 생성**한다.
> 핵심 학습 목표 : ① 역할 분리(기획 vs 생성) ② 아이디어 → 4컴포넌트(화면/상태/입력/결과) 분해 ③ 재현되는 디자인 프롬프트 작성법.
> 🔑 방식 : 클로드코드는 **design-brief.md + 복붙용 프롬프트**까지, 최종 시각물은 **클로드 디자인(Artifacts)** 이 만든다.
> ⛔ 자동 실행 금지. 단계마다 사용자 답을 받은 뒤에만 다음으로.

## 진행 흐름 (3 단계)

```
1단계  소개  — 역할 분리(기획/생성) + Before/After + 연결 소스           ⏸ 답 대기
2단계  실행  — 4컴포넌트 분해 → design-brief.md + 프롬프트 산출 → 핸드오프  ⏸ 답 대기
3단계  활용  — 내 아이디어 1개를 기획→프롬프트→클로드 디자인으로           ⏸ 선택
```

## 1단계 · 소개

```
🎨 Claude Design 프로토타입 실습을 시작합니다.

🎯 하는 일 — 역할을 둘로 나눈다
  · 클로드코드(기획) : 아이디어를 4컴포넌트로 분해 → 기획서(design-brief.md)
                       + "클로드 디자인에 그대로 붙여넣을 프롬프트" 를 만든다.
  · 클로드 디자인(생성) : 그 프롬프트를 받아 클릭 가능한 프로토타입(Artifact)을 만든다.
  "디자이너·개발자 없이, 기획은 코드가·시각물은 디자인이."

⏱ Before vs After
  · Before : 기획서 → 디자이너 요청 → 개발 대기 → 며칠 뒤 클릭 = 수일
  · After  : 아이디어 1줄 → (코드) 기획+프롬프트 → (디자인) 붙여넣기 → 클릭 = 5분
```

```

```
  | # | 항목 | 용도 | 상태 |
  |---|---|---|---|
  | ① | 클로드코드(이 세션)     | 기획 → design-brief.md + 프롬프트 | ✅ (기본) |
  | ② | 클로드 디자인(Artifacts) | 프롬프트 → 프로토타입 생성        | ✅ (claude.ai) |
  | ③ | 9-1 캠페인 카드          | 캠페인 랜딩 기획 입력(옵션)       | {✅ / ❌} |
  | ④ | Figma MCP(use_figma)     | 디자이너 핸드오프(옵션)           | {✅ / ❌} |

  💡 핸드오프 방식 : 클로드코드는 "프롬프트까지" 책임. 시각물은 클로드 디자인이 생성.
     별도 도구 없이도 재현 가능(프롬프트 텍스트만 있으면 됨).

📐 **왜 역할을 나누나?** → 기획(무엇을·어떤 흐름으로)은 맥락·데이터를 쥔 **클로드코드**가 잘하고,
  시각 완성도(레이아웃·인터랙션·디자인 톤)는 **클로드 디자인(Artifacts)** 이 강하다.
  둘을 잇는 매개가 **잘 쓰인 프롬프트 + 기획 md** 다. 9-1 캠페인 카드를 기획 입력으로 받을 수도 있다(9-1 → 9-2).

💡 핵심 포인트
  · 산출의 실체 = ① design-brief.md (기획서) + ② paste-prompt (클로드 디자인용 복붙 프롬프트)
  · 프롬프트엔 화면·상태·입력·결과 + 톤(모바일 375px·다크모드)까지 명시 → 재현성 확보
  · sample-output/index.html 은 "클로드 디자인이 이 정도로 뽑아준다"는 완성 기준 레퍼런스

⏸ 여기까지 이해되셨으면 "2단계" 라고 답해 주세요.

## 2단계 · 실행 (기획 → 프롬프트 → 핸드오프)

### 2-0 · 사전 점검
클로드 디자인은 claude.ai Artifacts 라 별도 설치가 없다. 9-1 카드·Figma 는 옵션.
이 단계에서 클로드코드가 만드는 것은 **파일 2개(기획 md + 프롬프트)** 이고, 시각물은 다음에 클로드 디자인이 만든다.

### 2-1 · 작동 구조 (아이디어 → 4컴포넌트 → 프롬프트)
```
  [아이디어 한 줄]
    → 4컴포넌트 분해
        · 화면 (screen)  : 몇 개의 화면/스텝인가 (예: 문항 → 결과)
        · 상태 (state)   : 진행 인덱스, 선택값, 결과 분기
        · 입력 (input)   : 라디오 / 슬라이더 / 텍스트
        · 결과 (result)  : 조건부 추천 페이지 분기
    → [클로드코드] design-brief.md 작성 (분해 + 카피 + 분기 규칙)
    → [클로드코드] paste-prompt 작성 (클로드 디자인에 그대로 붙여넣을 지시문)
        · 화면·상태·입력·결과 + 톤(모바일 375px·다크모드·단일 파일) 명시
    ──────── 핸드오프 ────────
    → [클로드 디자인] 프롬프트 붙여넣기 → 클릭 가능한 프로토타입(Artifact) 생성
    → (옵션) use_figma 로 Figma 핸드오프
```

📐 4컴포넌트 분해 표 (예시 : "신학기 모공 진단 미니 앱")

| 컴포넌트 | 이 아이디어에서 |
|---|---|
| 화면 | ① 3문항 진단 → ② 결과 추천 (2 화면) |
| 상태 | 현재 문항 index (0~2) · 답변 배열 · 점수 합산 |
| 입력 | 문항당 라디오 3택 |
| 결과 | 점수 구간별 결로아 토너 추천 분기 (모공/수분/민감 3종) |

📄 에이전트 정의에서 새로 볼 부분 (`agents/part9-expansion/claude-design-prototype.md`):
  ① 4컴포넌트 분해 규칙 — 아이디어를 화면/상태/입력/결과로 쪼개는 것이 기획의 전부
  ② 프롬프트 핸드오프 — 코드는 프롬프트까지, 시각물은 클로드 디자인이
  ③ marketing-calendar-builder 연결 — 9-1 캠페인 카드를 기획 입력으로

### 2-2 · 데이터 준비 (무엇을 만들지 고르기)

```
📊 입력 모드:

  [A] 내장 예시 프롬프트 3종 중 택 1 ⭐ 추천 · 촬영 기본
      sample-data/prototype-brief.md 의 3개 중 하나를 아이디어로 사용한다.
        1) 진단형 미니앱 : "신학기 모공 진단 미니 앱: 3문항 답 → 결로아 토너 추천 페이지"
        2) 랜딩 프로토타입 : "21일 챌린지 신청 랜딩 : 히어로 + 혜택 3 + 신청 폼"
        3) 인터랙티브 계산기 : "토너 리필 절약 계산기 : 사용량 입력 → 연간 절약액 표시"
      → 고르면 그 아이디어로 기획.md + 프롬프트를 만든다.

  [B] 내 아이디어 한 줄 직접 입력
      "___ 를 만들고 싶어" 한 줄 → 4컴포넌트 분해 후 기획.md + 프롬프트 산출.
      9-1 에서 만든 캠페인 카드를 붙여넣어도 됨 (캠페인 랜딩 기획으로 변환).
```

⏸ 1/2/3 또는 내 아이디어 한 줄을 주시면 2-3 에서 기획·프롬프트를 만듭니다.

### 2-3 · 실제 실행 (기획 → 프롬프트 산출 → 클로드 디자인 핸드오프)
```
▶ 고른 아이디어로 지금 진행:

  1. [클로드코드] 4컴포넌트 분해를 화면에 출력 (화면/상태/입력/결과)   → 이해 확인
  2. [클로드코드] design-brief.md 생성
       → outputs/prototypes/{이름}/design-brief.md
       · 분해 + 문항/섹션 스크립트 + 결과 분기 규칙 + 카피 + 톤 지침
  3. [클로드코드] paste-prompt.md 생성 (클로드 디자인 복붙용)
       → outputs/prototypes/{이름}/paste-prompt.md
       · "다음 사양의 단일 파일 프로토타입을 만들어줘 : 화면… 상태… 입력… 결과…
          모바일 375px·다크모드·의존성 0·클릭 검증 1분" 형태로 완결된 지시문
       · sample-data/paste-prompt-sample.md 를 골격 참고
  4. ──── 핸드오프 ────
       claude.ai(클로드 디자인)에서 새 대화 → paste-prompt.md 내용을 붙여넣기
       → Artifact 로 클릭 가능한 프로토타입 생성 → 미리보기에서 바로 클릭 검증
       (촬영 재현 : sample-output/index.html 이 "이 정도로 나온다"는 결과 레퍼런스)
  5. (옵션) use_figma 로 Figma 핸드오프 · Discord 에 URL/스크린샷 공유

  ✅ 성공 기준 : 붙여넣은 프롬프트만으로 클로드 디자인이 sample-output 수준의
     클릭 가능한 프로토타입을 재현한다 = 프롬프트가 잘 쓰였다는 뜻.
```

⏸ 프롬프트를 클로드 디자인에 붙여 프로토타입을 확인한 뒤 "3단계" 라고 답해 주세요.
