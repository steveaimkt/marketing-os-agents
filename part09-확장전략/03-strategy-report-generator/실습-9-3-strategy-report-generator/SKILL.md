# Part 9 / 9-3 · 마케팅 전략 보고서 생성 실습 : 소개 → 실행 → 활용

> GA4·광고·VoC·LTV·경쟁사 데이터를 **코드가 집계·기획하고 → 클로드 디자인이 임원용 시각 리포트로 완성**한다.
> 핵심 학습 목표 : ① 에이전트 5~6개 병렬 호출(최상위 오케스트레이션) ② 5장 구조 매핑 ③ 집계→기획md→디자인 핸드오프 + 외부공유 마스킹.
> 🔑 방식 : 클로드코드는 **report-content.md(5장 집계) + 디자인 프롬프트**까지, 임원용 시각물은 **클로드 디자인(Artifacts)** 이 만든다.
> ⛔ 자동 실행 금지. 단계마다 사용자 답을 받은 뒤에만 다음으로.

## 진행 흐름 (3 단계)

```
1단계  소개   역할 분리(집계·기획/시각화) + 연결 소스 + 핵심 포인트          ⏸
2단계  실행   사전 점검 → 5~6 집계 → 5장 md+프롬프트 → 클로드 디자인 핸드오프  ⏸
3단계  활용   내 업무 적용법 + 산출물 + Part 9 마무리 + Part 10               ⏸
```

## 1단계 · 소개

```
📊 마케팅 전략 보고서 생성 에이전트가 하는 일 — 역할을 둘로 나눈다

  · 클로드코드(집계·기획) : 5~6개 분석 에이전트를 병렬 호출 → 5장 구조로 정리한
                            report-content.md + "임원 1p 시각화용 디자인 프롬프트" 를 만든다.
  · 클로드 디자인(시각화) : 그 md/프롬프트를 받아 임원용 1페이지 시각 리포트(Artifact)를 만든다.
  "분기 보고서 만드느라 데이터 5곳을 밤새 긁지 않는다. 집계는 코드가·시각물은 디자인이."

  · Before : GA4·광고·VoC·LTV·경쟁사 각각 열어 캡처 → PPT 30장 수작업 = 2~3일
  · After  : 명령 1줄 → 5~6개 병렬 집계 → 5장 md+프롬프트 → 클로드 디자인 시각 리포트 = 5분
```

🔌 연결 소스 (실제 의존만)

```
strategy-report-generator  ← 최상위 오케스트레이터 (집계·기획)
  ├─ Agent(ga4-analyzer)               ← 트래픽·전환 추세
  ├─ Agent(3media-integrated-reporter) ← 3매체 광고 성과 (그 자체가 하위 3개 호출)
  ├─ Agent(voc-analyzer)               ← 고객 페인 포인트
  ├─ Agent(ltv-analyzer)               ← LTV 세그먼트 변화
  ├─ Agent(competitor-monitor)         ← 경쟁 환경 변화
  ├─ [산출] report-content.md + 디자인 프롬프트
  ├─ 클로드 디자인(Artifacts)          ← 임원 1p 시각 리포트 생성 (핸드오프)
  └─ Notion                            ← 상세 5장 다중 페이지 발행
```

💡 핵심 포인트 : 이 에이전트는 **에이전트 5~6개를 병렬 호출하는 최상위 오케스트레이션**이다.
  6-4(3매체) 가 에이전트 3개를 호출했다면, 9-3 은 그 6-4 를 **포함해** 5~6개를 부른다 (2단 오케스트레이션).
  집계 결과는 5장 report-content.md 로 모으고, **임원용 시각 완성도는 클로드 디자인에 프롬프트로 넘겨** 얻는다.
  선행 산출물이 있으면 그대로 쓰고, 없으면 내장 픽스처로 폴백해 단독 촬영 가능. Part 10 자연어 라우팅의 직전 예고편.

📐 **왜 역할을 나누나?** → 데이터 집계·인사이트·구조화(무엇을 담을지)는 맥락을 쥔 **클로드코드**가,
  임원 보고용 시각 완성도(레이아웃·강조·톤)는 **클로드 디자인**이 강하다. 매개는 **report-content.md + 프롬프트**.
  (기존 html-report-template 스킬로 바로 HTML 을 뽑는 경로도 가능하지만, 이 클립은 클로드 디자인 핸드오프를 학습한다.)
