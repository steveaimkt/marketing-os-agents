# 실습 1-2. 팀 역할 점검

## Step 1. 30개 에이전트 카탈로그 확인
```bash
ls marketing-os/agents/
# → orchestrator.md + part3 ~ part9 폴더 각각의 에이전트
```

## Step 2. 한 에이전트의 의존성 분석
```bash
claude "$(cat <<'EOF'
landing-copy 에이전트는 어느 다른 에이전트·스킬·MCP를 호출하나?
의존성을 트리 형태로 그려줘.
EOF
)"
```

기대 결과:
```
landing-copy
├── Skill: brand-voice
├── Agent: voc-analyzer (페인 포인트 입력)
├── Agent: ad-reference-collector (경쟁사 카피 패턴)
├── MCP: claude_ai_Notion (DB 쿼리·페이지 생성)
└── Skill: quality-review-6axis (출력 직전 검수)
```

## Step 3. 전체 의존 그래프 그리기
```bash
claude --agent claude-design-prototype "$(cat <<'EOF'
agents/ 폴더 전체를 읽고 에이전트 간 의존성을 인터랙티브 그래프로 시각화해줘.
산출물: outputs/prototypes/agent-graph/index.html (Vis.js 또는 D3)
EOF
)"
open outputs/prototypes/agent-graph/index.html
```

## 응용
- 새 에이전트를 추가할 때 의존성을 미리 설계하고 그래프 갱신
