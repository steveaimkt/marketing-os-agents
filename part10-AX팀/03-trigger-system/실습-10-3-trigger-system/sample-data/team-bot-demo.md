# 폰 라이브 시연 대본 (team-bot-demo) — 팀 공용 봇 도착 장면

> 용도 : 2-4 B-4 에서 실제 폰 시연이 어려울 때, 이 대본으로 "팀원 요청 → 오케스트레이터 라우팅 → 폰 회신" 흐름을 그대로 재현.
> 브랜드 : 결로아(가상) · 봇 페르소나 = 트루먼(오케스트레이터) · 채널 = #팀-마케팅봇 (requireMention: true)
> ⚠️ 수치는 가상 픽스처. 실제 가동 시 값은 달라진다.

---

## 장면 1 · 팀원 A(준서, 퍼포먼스) 요청

```
📱 준서 (#팀-마케팅봇)
   @트루먼 지난주 메타 광고 ROAS 뽑아줘

🤖 트루먼 (스레드 회신)
   📈 지난주 Meta 광고 (6/23~6/29)
   · 통합 ROAS 2.14 (전주 1.88 ▲ +14%)
   · TOP  : 21일 챌린지 전환 · ROAS 3.80
   · BOTTOM: 브랜드 인지 리치 · ROAS 0.43 (목적=도달, 단독판단 주의)
   📎 meta-roas-2026-06-29.html   🔗 Notion 아카이브
   → meta-ads-analyzer 로 라우팅 · 읽기 전용
```

## 장면 2 · 팀원 B(하나, 콘텐츠) 요청 — 동시 도착

```
📱 하나 (#팀-마케팅봇)
   @트루먼 경쟁사 신규 영상 정리해줘

🤖 트루먼 (스레드 회신)
   🎬 경쟁사 신규 콘텐츠 (최근 7일 · 3사)
   · A사 "모공 3주 챌린지" 릴스 3편 (조회 급상승)
   · B사 성분 비교 카드뉴스 5장
   · C사 변화 없음
   💡 우리도 21일 챌린지 릴스 대응 권장
   → competitor-monitor 로 라우팅
   ※ 두 요청(준서·하나)이 같은 세션에 도착 → 각 전문가로 병렬 라우팅 → 각자에게 회신
```

## 장면 3 · 승인이 필요한 요청 (권한 릴레이 + 승인 게이트)

```
📱 하나
   @트루먼 이 경쟁사 대응 콘텐츠, 인스타에 예약 발행해줘

🤖 트루먼
   🛑 발행은 승인 게이트 작업입니다.
   초안을 #marketing-approvals 에 올렸어요 → ✅ 반응 시 Buffer 예약, ❌ 시 취소.
   (발행·발송·삭제·예산은 자동 실행하지 않습니다 — 보안 고려 ②)

📱 (관리자 스티브 또는 신뢰 팀원) → #marketing-approvals 에서 ✅
🤖 트루먼 → 예약 완료 · 6/30 19:00 인스타 발행 큐잉
```

---

## 시연에서 반드시 짚을 3가지

1. **입구 1개** : 팀원 A·B 가 서로 다른 요청을 던져도 전부 같은 세션(트루먼) 하나로 도착.
2. **라우팅** : 오케스트레이터가 각각 meta-ads-analyzer / competitor-monitor 로 자동 분배.
3. **승인 게이트** : 조회는 자동, 발행은 ✅ 후에만 — 팀에 열어도 민감 작업은 이중 방어.
