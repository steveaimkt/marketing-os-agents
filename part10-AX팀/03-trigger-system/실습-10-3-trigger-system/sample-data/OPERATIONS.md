# 결로아 마케팅봇(트루먼) 사용설명서 — OPERATIONS.md (팀용 템플릿)

> 이 문서는 **사람용 매뉴얼**입니다. 봇이 매 메시지 자동 로드하지 않습니다.
> 실제 봇 행동은 프로젝트의 `CLAUDE.md` + `agents/orchestrator.md` 가 규정합니다.
> 배포 위치 : `~/.claude/channels/discord/OPERATIONS.md` (이 파일을 복사해 우리 팀 값으로 채우기)

## 1. 봇은 누구인가
- 이름 : 트루먼 (오케스트레이터 = 입구 1개 · 두뇌 1개)
- 하는 일 : 팀 요청을 받아 30개 전문 에이전트에게 분배 → 결과를 회신
- 창구 : #팀-마케팅봇 (공용 · @트루먼 멘션 필요) / 봇 DM (개인·외부)

## 2. 자주 쓰는 발화 (그대로 복사해서 써도 됨)
| 하고 싶은 일 | 이렇게 말하면 됨 |
|---|---|
| 지난주 광고 성과 | `@트루먼 지난주 메타 광고 ROAS 뽑아줘` |
| 3매체 통합 리포트 | `@트루먼 이번주 3매체 통합 리포트` |

| CS 답변 초안 | `@트루먼 이 문의 답변 초안 만들어줘` (내용 첨부) |

## 3. 권한 흐름 (무엇이 자동이고 무엇이 승인인가)
```
조회 (성과·리서치·시트 읽기)      → 자동 실행
발행/발송 (뉴스레터·SNS·CS 답변)  → #marketing-approvals 에서 ✅ 후 확정
삭제 · 광고 예산 변경             → 봇이 수행하지 않음 (금지)
```
> ⚠️ 채널에 답장할 수 있으면 도구 승인/거부(✅/❌)도 할 수 있습니다. 발행 승인은 신뢰 팀원만.

## 4. 팀 규칙 (우리 팀이 정해서 채우기)
- 발행 승인 가능자 : 스티브(관리자) · 하나(콘텐츠 리드)
- 광고 예산 변경 : 봇에게 요청 금지 — 관리자가 직접
- 고객 개인정보 시트 : 외부 프리랜서에게 노출 금지
- 요청 채널 : 성과·리서치 = #팀-마케팅봇 / 개인 실험 = 봇 DM

## 5. 응급 명령 (관리자용)
| 상황 | 명령 |
|---|---|
| 새 팀원 추가 | (팀원 메시지→코드) → `/discord:access pair <코드>` |
| 접근 잠금 | `/discord:access policy allowlist` |
| 봇 오프라인 | 프로젝트 폴더에서 `claude --channels plugin:discord@claude-plugins-official` 재시작 |
| 토큰 노출 의심 | Discord Reset Token → `/discord:configure <새 토큰>` |
| 상시 가동 | `caffeinate -dis` + `--channels` 세션 유지 |

## 6. 알아둘 것 (오해 방지)
- 봇은 **관리자 PC 의 세션 하나**로 돕니다. 세션이 꺼지면 봇도 멈춥니다.
- 팀원 요청도 **관리자 계정·인증**으로 실행됩니다(단일 인증 공유).
- iCloud 경로 프로젝트는 정기 cron 이 실패할 수 있어, 정기 작업은 클라우드 루틴을 씁니다.
