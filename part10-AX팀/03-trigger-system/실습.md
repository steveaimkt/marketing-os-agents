# 실습 1-3 — 대단원: 트리거 하나로 전체 가동

## Step 1. 디스코드 봇 슬래시 명령 등록
```bash
# 사전: discord-bot/slash-commands.md의 JSON을 등록
source .env
curl -X PUT \
  -H "Authorization: Bot ${DISCORD_BOT_TOKEN}" \
  -H "Content-Type: application/json" \
  -d @discord-bot/slash-commands.json \
  "https://discord.com/api/v10/applications/${DISCORD_APPLICATION_ID}/commands"
```

## Step 2. 인바운드 리스너 시작
```bash
# Option A: Hook 방식 (강의 권장)
cat ~/.claude/settings.json | jq '.hooks'
# UserPromptSubmit hook이 등록되어 있는지 확인

# Option B: 봇 서버 직접 실행
node discord-bot/server.js &  # 또는 Cloudflare Worker 배포
```

## Step 3. 첫 대단원 실행
**디스코드 #marketing-os 채널에서**:
```
/ax-team-run weekly
```

또는 터미널:
```bash
claude "/ax-team-run weekly"
```

## 예상 결과 (15~20분 후)

디스코드에 4개 메시지 도착:
1. 🚦 시작 알림 — "Phase 1 시작 / 4개 리서치 에이전트 호출"
2. 📡 Phase 2 시작 — "분석 6개 에이전트 병렬"
3. 🔧 Phase 3 진행 중 — "합성 4개"
4. 🏢 **종합 embed** — 모든 결과 통합 (위 [`commands/ax-team-run.md`](../../../commands/ax-team-run.md) 참조)
5. 📋 #marketing-approvals 채널 — content-publisher 큐잉 승인 요청

## Step 4. 자동화 등록 (24시간 가동)
```bash
# launchd plist 3개 설치
cp automation/com.marketing-os.daily-briefing.plist ~/Library/LaunchAgents/
cp automation/com.marketing-os.weekly-report.plist ~/Library/LaunchAgents/
cp automation/com.marketing-os.hourly-ad-check.plist ~/Library/LaunchAgents/

launchctl load ~/Library/LaunchAgents/com.marketing-os.*
launchctl list | grep marketing-os
# → 3개 모두 LOADED
```

또는 crontab:
```bash
crontab automation/*.cron
crontab -l
```
