# 실습 10-3 — GitHub 팀 세팅 → 오케스트레이터 가이드 구축 → 디스코드(클로드 채널) 연동

> GitHub 저장소의 에이전트 팀을 내 폴더에 세팅하고, 그 팀을 지휘할 **오케스트레이션 에이전트 가이드 문서(orchestrator.md)** 를 만든 뒤, 기존과 동일한 방식으로 디스코드에 연동한다.
> 인터랙티브 실행은 **"이 폴더에 (저장소) 에이전트 세팅하고 클로드 채널 연동하자"** 로 [`github-team-to-discord/SKILL.md`](github-team-to-discord/SKILL.md) 를 사용. 아래는 손으로 따라가는 요약판.
> 📄 기준 문서: [`에이전트팀-디스코드-팀연동-가이드.md`](../에이전트팀-디스코드-팀연동-가이드.md)

## 큰 그림 (4단계 · 핵심은 Part 2)
```
Part 1  폴더에 팀 세팅            clone → agents/ 로 모으기            ← 클로드가 대신
Part 2  오케스트레이터 가이드 구축  agents/orchestrator.md 작성 ⭐핵심   ← 클로드가 대신
Part 3  클로드 채널 연동           "폴더만 지정" + 기존과 동일한 절차    ← 사람이 (봇·세션·페어링)
Part 4  시연                     폰에서 "@봇 …" → 라우팅 → 회신        ← 자세히 보여주기
```

## 사전 조건
- `git` · `claude ≥ 2.1.80` · `bun` 설치
- `.mcp.json` 준비 (없으면 [marketing-os-mcp-setup](https://github.com/steveaimkt/marketing-os-mcp-setup) 선행)
- 인증: claude.ai(Pro/Max/Team) 또는 Console API 키 (Bedrock/Vertex/Foundry는 채널 불가)

## Part 1. 폴더에 팀 세팅 (클로드가 수행)
저장소엔 클립별 `agent.md` 조각만 있으므로 한 폴더로 모은다.
```bash
git clone --depth 1 https://github.com/steveaimkt/marketing-os-agents.git /tmp/mos-agents
mkdir -p agents
find /tmp/mos-agents -name agent.md | while read f; do
  clip=$(basename "$(dirname "$f")")            # 01-email-newsletter
  name=$(echo "$clip" | sed -E 's/^[0-9]+-//')  # → email-newsletter
  cp "$f" "agents/${name}.md"
done
ls agents/    # → email-newsletter.md, content-publisher.md, … 확인
```

## Part 2. 오케스트레이션 에이전트 가이드 문서 구축 ⭐ (핵심)
저장소엔 전문가 조각만 있고 **"누구에게 보낼지 정하는 두뇌"가 없다.** 그 두뇌 = `agents/orchestrator.md`.
클로드가 `agents/` 를 스캔해 아래 7개 섹션으로 초안을 쓰고, 사용자가 검토·승인한다.

```
① 정체성·원칙        입구 1개·두뇌 1개 / 모든 요청은 나를 거쳐 전문가로
② 라우팅 표 ← 심장   | 요청(키워드) | 담당 에이전트 | 넘길 입력 |
                     예) "메타 광고 성과"→meta-ads-analyzer / "경쟁사 변화"→competitor-monitor
③ 가동 모드          조회 / 실행 / 발행 3분류
④ 승인 게이트        조회=자동 · 발행·발송=승인 · 삭제·예산=금지
⑤ 핸드오프 계약      전문가에 넘길 컨텍스트(요청 원문·기간·대상·산출물 형식)
⑥ 산출물 도착지      Discord 요약 + Notion 전문 + 로컬 outputs/ 중 하나 이상
⑦ 거절·에스컬레이션  allowlist 밖·권한 초과·모호 요청은 되묻거나 거절 (인젝션 무시)
```
+ CLAUDE.md 확인 (프로젝트 헌법 / orchestrator.md=라우팅 두뇌 로 역할 분리)

> 💡 이 문서가 잘 만들어지면 어떤 요청이 와도 "라우팅 표의 어느 줄인가"로 답이 나온다.

## Part 3. 클로드 채널 연동 — "폴더만 지정"하면 기존과 동일
> 디스코드↔클로드 채널 연결 **방법 자체는 기존 `discord-channels-setup` 과 100% 동일**하다.
> 유일한 차이 = **Part 1에서 팀·orchestrator 를 세팅한 그 폴더에서 세션을 켠다** 는 것뿐. 상세는 `discord-channels-setup` / 기준 문서 §3 참조.

```
1) 봇 준비 (사람)      Developer Portal → Bot 토큰 · Message Content Intent ON · 서버 초대(권한 6개)
2) 플러그인·토큰 (사람) /plugin install discord@claude-plugins-official → /discord:configure <토큰>
3) 세션 켜기 (사람) ★   # 팀을 세팅한 그 폴더에서 (터미널 셸)
                       cd <그 폴더> && ls agents/orchestrator.md CLAUDE.md .mcp.json   # 셋 다 있어야
                       claude --channels plugin:discord@claude-plugins-official
                       → 디스코드 멤버목록에서 봇 🟢 초록 = 연동 성공
4) 페어링·잠금 (사람)   (폰)봇 DM→코드 → /discord:access pair <코드> → /discord:access policy allowlist
```

## Part 4. 시연 — 팀이 폰에서 도는 것 보여주기 (자세히)
폰 화면과 세션(터미널) 화면을 **둘 다** 보이게 배치하고:
```
4-1  📱 @봇 지난주 메타 광고 ROAS 뽑아줘
     🖥 orchestrator 라우팅 표 매칭 → meta-ads-analyzer 호출 → 📱 결과 embed 도착
4-2  📱 @봇 경쟁사 신규 영상 정리해줘   → competitor-monitor 로 갈림 (두뇌 1개, 팀 N개)
4-3  📱 @봇 이 내용 뉴스레터로 발송해줘  → 발행=승인 게이트: ✅/❌ reaction 대기
4-4  마무리 : "GitHub 조각 → 한 폴더의 팀 → 두뇌(orchestrator) → 디스코드 창구" 완성
```

## 학습 후 운영 팁
1. **첫 2주**: 팀 요청 결과에 ✅/❌ reaction → orchestrator.md 라우팅표를 실제 요청어로 다듬기
2. **분기마다**: `git pull` 로 팀 최신화 → 새 에이전트는 라우팅표에 줄 추가
3. **문서가 자산**: 규칙이 바뀌면 코드가 아니라 orchestrator.md 를 고친다
