# 실습 1-1. AX vs 단일 에이전트 — 직접 비교

## Step 1. 단일 에이전트 가짜 시뮬레이션
```bash
claude "$(cat <<'EOF'
[비유 실습] 다음을 1개 에이전트가 하나의 컨텍스트에서 한꺼번에 한다고 가정하고
모든 워크플로를 너 혼자 다 수행해줘:

- 트렌드 스캔
- 경쟁사 모니터링
- 광고 점검
- 카피 10패턴 생성
- GA4 리포트

각 단계마다 토큰 사용량 추정도 함께.
EOF
)"
# → 결과: 매우 오래 걸리고 비용이 큼. 일부는 일관성 떨어짐
```

## Step 2. AX 팀으로 동일 작업
```bash
claude /daily-briefing  # 또는
claude --agent marketing-os-orchestrator "위 5개를 팀에 분배해서 진행"
# → 결과: 병렬로 빠름. 각 결과의 일관성·품질 우수
```

## Step 3. 비교 표 작성
직접 두 결과를 비교해 노션에 표 작성:
```bash
claude "두 실행 결과를 비교 표로 정리:
  - 소요 시간
  - 토큰 비용
  - 산출물 품질 (0~5)
  - 디버깅 용이성"
```

## 교훈 (학습자가 직접 확인)
- AX는 **소요 시간 ~30% 단축** (병렬)
- 토큰 비용 **~50% 절감** (필요한 컨텍스트만)
- 산출물 품질 향상 (각 영역 전문 에이전트가 처리)
