# 클립 1-1. AX란 무엇인가 — 에이전트 1개 vs 에이전트 팀

## 핵심 비교

| | 에이전트 1개 | **AX 팀** |
|---|---|---|
| 비유 | "만능 AI 한 명" | "마케팅 부서" |
| 구성 | 워크플로 30개를 한 에이전트에 욱여넣기 | 디렉터 1 + 전문가 30 |
| 토큰 | 매 호출마다 30개 정의 로드 → 폭증 | 디렉터는 라우팅만, 호출 시 전문가만 |
| 권한 | MCP 12개 모두 한 곳에 | 에이전트별 MCP 권한 분리 |
| 디버깅 | 어디서 실패했나? | 에이전트별 로그 |
| 패턴 | Claude Code 비권장 | **권장 패턴 (subagents + skills)** |
| UX | 봇 1개 | **봇 1개 (동일)** |

## 왜 UX가 같은가
사용자는 디스코드 채널에서 "/daily-briefing" 하나로 명령.
내부에서 디렉터가 알아서 3개 전문가에게 일을 분배 → 결과를 1개 메시지로 통합.

## 강의 핵심 메시지
**"단일 에이전트로 만들지 말 것. 처음부터 팀 구조로."**
