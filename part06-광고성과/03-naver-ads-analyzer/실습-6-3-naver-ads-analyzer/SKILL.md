# Part 6 / 6-3 · 네이버 검색광고 분석 실습 : 브랜드 vs 일반 분리

> 네이버 검색광고를 **브랜드 키워드와 일반 키워드로 분리 집계**해 합산 ROAS 의 착시를 깨는 에이전트를 가동한다.
> 핵심 학습 목표 : ① 분리 집계의 이유 ② 합산 지표의 착시 ③ MCP 부재 시의 대안 경로 (REST API).
> ⛔ 자동 실행 금지. 단계마다 사용자 답을 받은 뒤에만 다음으로.

## 진행 흐름 (3 단계)

```
1단계  소개   하는 일 + 연결 소스 (샘플 전용 안내)           ⏸ 확인
2단계  실행   작동 구조 + 데이터 준비 + 실제 실행 (착시 해부) ⏸ 답 대기
3단계  활용   내 업무 적용 + 실연동 경로 + 산출물 + 다음 클립 ⏸ 선택
```

---

## 1단계 · 소개

```
🟢 네이버 검색광고 분석 에이전트를 소개합니다.

본 흐름의 핵심 한 줄 :
  "브랜드 키워드를 섞어 평균 내면 모든 광고가 흑자처럼 보인다 → 그래서 분리한다"

⏱ Before vs After
  · Before : 광고 시스템 합산 ROAS 만 보고 "잘 되네" 오판 → 일반 키워드 적자 방치
  · After  : 브랜드/일반 자동 분리 → 어디서 벌고 어디서 새는지 즉시 보임
```

🔌 연결 소스 (실제 의존만)

```
  사용자 (기간 · 기본 최근 7일)
    │
    ├─ 📂 키워드 성과 데이터  ← 샘플 픽스처 (Naver Ads 공식 MCP 부재)
    │
    └─ 📨 Discord webhook     ← 분리 리포트 발송

  ⚠️ 네이버 검색광고는 공식 MCP 가 없습니다.
     → 본 실습은 샘플 데이터로 "분석 로직" 을 익히는 데 집중.
     → 실연동(본인 계정 REST API)은 3단계에서 별도 안내.
```

💡 핵심 포인트

```
  · 합산 ROAS 1 개 숫자에 속지 않기 = 이 클립의 전부
  · 브랜드 키워드 ROAS 가 높은 건 "광고를 잘해서" 가 아니다
    → 이미 살 사람이 자사명을 검색 · 방어 목적 입찰 (구조적 착시)
  · MCP 가 없어도 에이전트 정의(분석 규칙)는 그대로 재사용 가능
```

📐 **스킬? 에이전트?** → 공식 MCP는 없지만 '분석 규칙 + 산출(Discord)'을 가진 🤖 **에이전트**(수집부만 REST/샘플로 대체). 5요소 골격은 3-1과 같다.

⏸ 확인 받고 2단계로.


```
  1. 키워드 8개 로드
  2. 브랜드/일반 자동 분리                  → 판별 근거 표시
  3. 그룹별 집계 (시연 하이라이트):

  4. 일반 키워드 진단                       → "토너 추천" 97만 지출 · ROAS 0.59 최악
       개선 옵션 : 입찰가 하향 / 소재 교체 / LP 개선 / 축소
  5. Discord 분리 리포트 embed
```

### 오래 쓰는 법 (적용 후 정착)

🎯 내 업무 적용법

```
  1) 합산 지표 불신 습관화
     매체가 보여주는 "전체 ROAS" 는 항상 브랜드 키워드가 섞인 평균.
     → 첫 질문은 늘 "브랜드 빼면 얼마?"

  2) 분리 규칙 한 줄로 운영
     브랜드 = 자사명·제품명 포함 키워드. .env 에 자사명 목록만 등록하면 자동 분류.

  3) 일반 키워드 = 진짜 광고 실력
     일반 ROAS 가 손익분기(보통 1.0~목표값) 미만이면 입찰가·소재·LP 중 하나가 범인.
     → 개선안은 5-2 카피 10패턴 에이전트로 즉시 제작.

  4) 본인 계정 실연동 (REST API 경로)
     manage.searchad.naver.com → 도구 → API 관리
       → API_KEY · SECRET_KEY · CUSTOMER_ID 발급 → .env 등록
       → REST API 서명 호출 (Bash) 로 실데이터 수집
       → 에이전트 정의(분석 규칙)는 샘플과 100% 동일하게 재사용
     ※ 공식 MCP 가 없으므로 수집부만 REST 직접 호출로 대체하는 방식.
```
