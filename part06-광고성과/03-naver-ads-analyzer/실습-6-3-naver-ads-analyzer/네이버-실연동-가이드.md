# 네이버 실연동 가이드 (6-3 부록)

> 6-3 `실습-6-3-naver-ads-analyzer` 스킬의 실연동·설치 레퍼런스. 시연(SKILL.md)과 분리해 보관.
> 전체 표준 설치 흐름은 `/mcp설치-네이버` 스킬과 동일. 여기는 6-3 맥락의 발췌.

## A · 네이버 3종 도구 설명 & 설치

> ⚠️ **"네이버 공식 MCP" 는 없다.** 검색광고는 REST 전용(MCP 없음), 검색·데이터랩 MCP는 **커뮤니티(비공식)** 패키지다.

### ① 네이버 검색광고 API (SearchAd) — 본 클립의 주역

- **무엇** : 광고 성과(노출·클릭·광고비·전환·ROAS) + 키워드 검색량(`/keywordstool`) + 예상 입찰가.
- **MCP** : ❌ 없음 → **REST 직접 호출**(HMAC-SHA256 서명). 구현체 `scripts/naver_searchad.py`.
- **설치** :
  1. `manage.searchad.naver.com` → **도구 → API 사용 관리**
  2. `API_KEY` · `SECRET_KEY` · `CUSTOMER_ID` 발급 → `.env` 등록
     ```
     NAVER_SEARCHAD_API_KEY=…  NAVER_SEARCHAD_SECRET_KEY=…  NAVER_SEARCHAD_CUSTOMER_ID=…
     ```
  3. 검증 : `python3 scripts/naver_searchad.py GET /ncc/campaigns` → `HTTP 200`
- **한계** : `/stats` 성과는 **최근 92일만**. 과거는 광고주센터 「대용량 다운로드 보고서」 CSV → `scripts/naver_ad_report.py`.

### ② 네이버 검색 MCP (naver-search-mcp) — 커뮤니티, 선택

- **무엇** : 블로그·뉴스·웹 검색 + 데이터랩(검색어트렌드/쇼핑인사이트). (광고 분석엔 불필요, 트렌드 리서치용)
- **출처** : `@isnow890/naver-search-mcp` (Smithery 배포). **비공식**.
- **설치** :
  1. `developers.naver.com` → 애플리케이션 등록 → **검색 / 데이터랩** API 선택 → `CLIENT_ID`·`CLIENT_SECRET`
  2. Smithery(`smithery.ai/server/@isnow890/naver-search-mcp`) 또는 npx 로 MCP 등록
  3. `.mcp.json` 에 `env: { NAVER_CLIENT_ID, NAVER_CLIENT_SECRET }`
- **주의** : ②③은 같은 `developers.naver.com` Client 키를 공유(검색+데이터랩 API 모두 신청해두면 됨).

### ③ 네이버 데이터랩 MCP (naver-datalab-mcp-server) — 커뮤니티, 본 시스템 채택

- **무엇** : 검색어 트렌드(시계열·성별·연령·디바이스) + 쇼핑인사이트. 검색광고의 검색량을 **추세로 교차검증**.
- **출처** : `naver-datalab-mcp-server` (npm v1.1.2, `icraft2170/naver-data-mcp-server`). **비공식**.
- **설치** :
  1. `developers.naver.com` → 애플리케이션 등록 → **데이터랩(검색어트렌드)+(쇼핑인사이트)** → `CLIENT_ID`·`CLIENT_SECRET`
     ```
     NAVER_CLIENT_ID=…       # 긴 값
     NAVER_CLIENT_SECRET=…   # 짧은 값  (401 시 두 값 순서 뒤집기)
     ```
  2. `.mcp.json` 등록 (bash source .env 패턴) :
     ```json
     "naver-datalab": { "command":"bash",
       "args":["-c","set -a; source '<프로젝트>/.env'; set +a; exec npx -y naver-datalab-mcp-server"] }
     ```
  3. 검증 : `POST https://openapi.naver.com/v1/datalab/search` (헤더 `X-Naver-Client-Id/Secret`) → `HTTP 200`

### 한눈에

| 도구 | 종류 | 발급처 | 키 | 주는 것 |
|---|---|---|---|---|
| ① 검색광고 API | REST(공식, MCP 없음) | manage.searchad.naver.com | API/SECRET/CUSTOMER | 광고성과·검색량·입찰가 |
| ② 검색 MCP | 커뮤니티 MCP | developers.naver.com | CLIENT_ID/SECRET | 블로그/뉴스 검색·데이터랩 |
| ③ 데이터랩 MCP | 커뮤니티 MCP | developers.naver.com | CLIENT_ID/SECRET | 검색트렌드·쇼핑인사이트 |

→ 6-3 실습은 ①(성과 분리)이 주역, ③(데이터랩)으로 교차검증. ②는 선택.

## B · 전체 구축 프로세스 (실제 진행 순서 그대로)

> 이 에이전트를 "샘플"이 아니라 **실데이터**로 돌리기까지 밟은 전 과정. 그대로 따라하면 재현된다.

### STEP 1 · 검색광고 API 연동 (REST 서명)

```bash
# 1) manage.searchad.naver.com → 도구 → API 사용 관리 → 3키 발급 후 .env 등록
#    NAVER_SEARCHAD_API_KEY / NAVER_SEARCHAD_SECRET_KEY / NAVER_SEARCHAD_CUSTOMER_ID
# 2) 서명 호출기는 scripts/naver_searchad.py (HMAC-SHA256, 의존성 0)
python3 scripts/naver_searchad.py GET /ncc/campaigns        # HTTP 200 = 인증 성공
python3 scripts/naver_searchad.py GET /keywordstool --params '{"hintKeywords":"전기톱,전동가위","showDetail":"1"}'
```
- 빈 `[]` = 광고 미집행 계정(인증은 정상). 캠페인이 있으면 `/ncc/adgroups` → `/stats` 로 성과 조회.
- **검증 포인트** : `/stats` 는 최근 92일만. 과거 데이터는 STEP 4-b 의 CSV import 로.
- **집행 여부 판별 (92일 벽 우회)** : `/stats` 가 0이어도 광고를 돌렸는지 확인하려면 비즈머니로 역산.
  ```bash
  python3 scripts/naver_searchad.py GET /billing/bizmoney                          # 현재 잔액
  python3 scripts/naver_searchad.py GET /billing/bizmoney/histories/charge \
    --params '{"searchStartDt":"2024-11-01","searchEndDt":"2026-06-19"}'           # 충전 이력
  # (충전 합계 − 현재 잔액 − 환불) ≈ 실제 소진 광고비.  캠페인 totalChargeCost 0 + 소진>0 = 삭제된 캠페인에서 집행됨
  ```

### STEP 2 · 데이터랩 MCP 연동 (교차검증용)

```bash
# 1) developers.naver.com → 애플리케이션 등록 → 데이터랩(검색어트렌드)+(쇼핑인사이트)
#    → CLIENT_ID(긴값) / CLIENT_SECRET(짧은값) → .env (NAVER_CLIENT_ID / NAVER_CLIENT_SECRET)
#    ⚠️ 401 나면 두 값 순서 뒤집기 (ID=긴값, SECRET=짧은값)
# 2) .mcp.json 에 naver-datalab 등록 (bash source .env 패턴) → Claude 재시작 시 도구 노출
# 3) 검증 (재시작 없이 직접):
curl -s -X POST https://openapi.naver.com/v1/datalab/search \
  -H "X-Naver-Client-Id: $NAVER_CLIENT_ID" -H "X-Naver-Client-Secret: $NAVER_CLIENT_SECRET" \
  -H "Content-Type: application/json" \
  -d '{"startDate":"2025-06-01","endDate":"2026-05-31","timeUnit":"month","keywordGroups":[{"groupName":"전기톱","keywords":["전기톱"]}]}'
# → HTTP 200 + ratio 배열이면 성공
```

### STEP 3 · 시장 수요 리포트 (검색광고 검색량 × 데이터랩 트렌드 교차검증)

```bash
python3 scripts/naver_market_report.py --seeds 전기톱,전동가위 --date 2026-06-19
```
- 흐름 : `/keywordstool` 연관어 확장 → **관련성 필터**(제품군 토큰, NC·무선이어폰 노이즈 제거)
  → 검색량 정렬 → TOP5 를 DataLab 트렌드로 **모멘텀 검증**(최근3M vs 직전3M)
  → ✅검증된수요 / ⚠️정점 / 🟢안정 판정 → HTML 리포트.
- 산출 : `outputs/{날짜}/naver-market/craftvolt-naver-report-{날짜}.html`

### STEP 4 · 광고 성과 ROAS 분리 리포트 (6-3 핵심)

```bash
# 4-a) 실습(데모) = 샘플 (결로아 → 합산154 / 브랜드741 / 일반71%)
python3 scripts/naver_ad_report.py sample-data/naver-ads-sample-report.csv --brand 결로아
# 4-b) 실전 = 본인 CSV (광고주센터 → 보고서 → 대용량 다운로드 보고서 → 기간지정 CSV)
python3 scripts/naver_ad_report.py <본인보고서.csv> --brand 크래프트볼트,craftvolt
```
- 임포터가 인코딩(utf-8-sig/cp949)·구분자(tab/comma)·컬럼명 **자동 감지** → 브랜드/일반 분리 → 합산 착시 해부 + 적자 키워드 진단.

### STEP 5 · Discord 발송 (⛔ 승인 게이트)

```bash
python3 scripts/naver_market_report.py --send --webhook <URL>     # 시장 리포트
python3 scripts/naver_ad_report.py <csv> --brand 결로아 --send --webhook <URL>  # ROAS 리포트
```
- `--send` 없으면 미리보기. 성공 = **HTTP 204**. 외부 발행이라 첫 발송은 사용자 승인 후.

## 함정·디버깅 메모 (실제로 부딪힘)

| 증상 | 원인 | 해결 |
|---|---|---|
| DataLab 401 | Client ID/Secret 순서 바뀜 | ID=긴값, Secret=짧은값 뒤집기 |
| Discord 403 / 1010 | Cloudflare가 `python-urllib` UA 차단 | 요청에 **User-Agent 헤더** 추가(반영됨) |
| 키워드 1위가 'NC·무선이어폰' | 연관어 확장 과도 | **관련성 필터**(제품군 토큰) |
| `/stats` 과거 0 | 92일 벽 + 광고 OFF | 대용량 다운로드 보고서 CSV → STEP 4-b |
| 브랜드/일반 안 맞음 | 쇼핑검색은 상품 기반 | 분리는 파워링크(검색광고) 기준 |
