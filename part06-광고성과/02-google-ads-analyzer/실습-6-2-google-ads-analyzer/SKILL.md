# Part 6 / 6-2 · Google Ads 분석 에이전트 실습 : 소개 → 실행 → 활용

> 검색 캠페인의 **키워드·검색어·품질점수를 3개 섹션으로 진단**하는 에이전트를 가동한다.
> 핵심 학습 목표 : ① 키워드 vs 검색어의 차이 ② 품질점수와 CPC ③ 제외 키워드 후보 도출.
> ⛔ 자동 실행 금지. 각 단계 사용자 답을 받은 뒤에만 다음으로.

## 진행 흐름 (3 단계)

```
1) 소개   이 에이전트가 하는 일 + 연결 소스 안내            ⏸ 답 대기
2) 실행   작동 구조 → 데이터 준비 → 실제 실행 시연          ⏸ 답 대기
3) 활용   내 업무에 적용하는 법 + 산출물 + 다음 클립        ⏸ 선택
```

---

## 1단계 · 소개

```
🔎 Google Ads 분석 에이전트 — "내가 산 키워드(keyword)와 고객이 친 검색어(search term)는
   다르다. 그 틈에서 돈이 샌다"를 2분 만에 진단합니다.

⏱ Before vs After
  · Before : 키워드 플래너 + 검색어 보고서 + 품질점수 컬럼 수기 정리... 익숙해도 1시간/주
  · After  : 명령 1줄 → 3섹션 진단 + 제외 키워드 후보까지 = 2분

🔌 연결 소스 (실제 의존만)
  | 항목 | 용도 | 상태 |
  |---|---|---|
  | Google Ads MCP (uvx) | 검색 성과 조회 (GAQL · 읽기 전용) | {✅ / ❌} |
  | Discord webhook      | 3섹션 진단 결과 발송              | {✅ / ❌} |

  점검 명령:
    claude mcp list 2>&1 | grep -i "google-ads"
    test -f "discord-bot/webhook-config.json" && echo "webhook ✅"

  ⚠️ Google Ads 공식 MCP는 Developer Token 신청 → 승인에 1~2일이 걸릴 수 있습니다.
     승인 대기 중이면 샘플 모드로 진행하세요 (촬영 권장 경로).

💡 핵심 포인트
  · 키워드 = 내가 입찰한 단어 / 검색어 = 고객이 실제로 입력한 문장 → 둘은 다르다
  · 품질점수가 낮으면 같은 노출에도 CPC가 오른다 (돈이 더 샌다)
  · 제외 키워드는 "후보"로만 제시 — 실제 적용은 사용자 (읽기 전용 + 승인 원칙)
```

📐 **스킬? 에이전트?** → 도구(Google Ads) + 산출(Discord 진단·HTML)이 있으니 🤖 **에이전트**. 리포트는 html-report-template **스킬** 공유. 5요소 골격은 3-1과 같다.

⏸ "다음" / "실행" 받으면 2단계로.

## 2단계 · 실행

### 2-0 · 사전 점검

```bash
claude mcp list 2>&1 | grep -i "google-ads"
test -f "discord-bot/webhook-config.json" && echo "webhook ✅"
```

❌ 항목이 있으면 : Google Ads는 ADC 인증(`gcloud auth application-default login`) +
`.mcp.json` uvx 등록. 승인 대기 시 샘플 모드로 건너뛰기 허용.

### 2-1 · 작동 구조 (3섹션)

```
[기간 (기본 최근 7일)]
  → 섹션 1 키워드 성과    : ROAS TOP + CPC 비효율 BOTTOM
  → 섹션 2 검색어 리포트  : 실제 검색어 vs 매칭 키워드 정합 (전환 0 검색어 색출)
  → 섹션 3 품질점수 경고  : 7 미만 키워드 (CPC 상승 위험) + 원인 추정
  → 제외 키워드 후보 3~5건 (높은 지출 + 전환 0)
  → Discord 발송 (적용은 사용자 수동 : 읽기 전용 원칙)
```

📄 에이전트 정의에서 새로 볼 부분 (`agents/part6-ads/google-ads-analyzer.md`):
  ① GAQL (Google Ads Query Language) : SQL 비슷한 조회 언어를 에이전트가 대신 작성
  ② 제외 키워드를 "후보"로만 제시 : 적용은 사용자 (읽기 전용 + 승인 원칙의 또 다른 형태)
  ③ 품질점수 경고에 원인 추정 동반 : 점수만 주지 않고 다음 행동 제시

### 2-2 · 데이터 준비

```
📊 데이터 모드:

  [A] 샘플 데이터 모드 ⭐ 추천 · 촬영 기본
      sample-data/google-ads-performance.json 로드.
      키워드 15 (품질점수 5·6점 경고 2건 포함) + 검색어 12 (제외 후보 4건 포함).
      경고·제외 후보 장면이 결정적으로 재현됨. ⚠️ 픽스처는 수정하지 않습니다.

  [B] 실계정 모드
      본인 Google Ads 검색 캠페인 라이브 조회 (GAQL · 읽기 전용).
      → 캠페인 선택 후 검색 캠페인 존재 확인.
```

⏸ A/B 답 받고 2-3으로.

### 2-3 · 실제 실행

```
▶ 에이전트 워크플로 단계별 실행:

  1. 데이터 로드
  2. 섹션 1 키워드 성과    → TOP : 브랜드 "결로아" (CPC 190 · 전환 71 · 가치 994,000 · 방어 효과)
                            BOTTOM : "토너" (CPC 760 · 전환 19 · 광범위 단일어)
  3. 섹션 2 검색어 정합 (시연 하이라이트) → 전환 0 검색어 색출:
       유통기한 지난거 / 영어로 / 피부과 모공 시술 (각각 왜 돈이 새는지 한 줄 설명)
       + 기회 발견 : "메디큐브 토너 비교" 검색어 (전환 7) → 비교 LP 아이디어
  4. 섹션 3 품질점수 경고  → "토너" 5점 (광범위 단일어·CPC 위험) ·
                            "21일 챌린지" 6점 (챌린지 전용 LP 부재) + 원인 추정
  5. 제외 키워드 후보 4건 정리 : 유통기한 / 영어로 / 피부과 시술 / 화장솜
                                → 적용 방법 안내 (사용자 수동)
  6. Discord 진단 결과 발송
```

⏸ 결과 확인 후 3단계로.

→ "건너뛰기" 시 아래 정착 가이드만.

### 오래 쓰는 법 (적용 후 정착)

```
🎯 내 업무에 적용하기

  1. 매주 월요일 자동 실행 → 지난주 전환 0 검색어를 제외 키워드 후보로 받아 검토
  2. 품질점수 7 미만 키워드 = 랜딩페이지 점검 신호 → 5-3 상세페이지 카피로 보강
  3. 경쟁사 비교 검색어 ("OO 비교") 발견 시 → 비교 LP 제작 기회로 전환
  4. 브랜드 키워드(CPC 낮고 전환 높음) 방어 예산을 우선 확보 — 잠식 방지

📊 산출물:
   · Discord : 3섹션 진단 (키워드 성과 + 검색어 리포트 + 품질점수 경고) + 제외 키워드 후보 4건

   ✅ 체크리스트:
   [ ] 키워드 vs 검색어 차이를 설명할 수 있음
   [ ] 품질점수가 CPC 에 미치는 영향 이해
   [ ] 제외 키워드를 에이전트가 직접 적용하지 않는 이유

🔁 다음 클립:

   B. 실계정 모드로 본인 검색어 리포트 진단
   C. 비교 LP 아이디어를 5-3 상세페이지로 제작
```
