# Part 6 / 6-1 · Meta 광고 분석 에이전트 실습 : 소개 → 실행 → 활용

> Meta 광고 성과를 **읽기 전용으로 가져와 ROAS 기준 TOP/BOTTOM 과 추세를 HTML 리포트**로 만든다.
> 핵심 학습 목표 : ① KPI 5종 (CTR·CPC·CPA·ROAS·전환율) ② 전주 비교 ③ 읽기 전용 안전 원칙.
> ⛔ 자동 실행 금지. 단계마다 사용자 답을 받은 뒤에만 다음으로.

## 진행 흐름 (3 단계)

```
1단계  소개  — 하는 일 + Before/After + 연결 소스 + 핵심 포인트     ⏸ 답 대기
2단계  실행  — 작동 구조 + 데이터 준비 + 실제 실행 (KPI → 리포트)   ⏸ 답 대기
3단계  활용  — 내 업무 적용법 + 산출물 + 다음 클립                   ⏸ 선택
```

---

## 1단계 · 소개

```
📈 Meta 광고 분석 에이전트 실습을 시작합니다.

🎯 하는 일
  "광고 관리자 들어가서 헤매지 않는다. 리포트가 아침에 와 있다."
  Meta 광고 성과를 읽기 전용으로 가져와 → KPI 5종 계산 → ROAS TOP/BOTTOM 선별
  → 전주 대비 추세까지 담은 HTML 리포트 1장으로 정리한다.

⏱ Before vs After
  · Before : 광고 관리자 접속 → 기간 설정 → 열 편집 → 엑셀 → 정리 = 약 40분/회
  · After  : 명령 1줄 (또는 매일 자동) → HTML 리포트 + TOP 3 알림 = 1분
```

🔌 연결 소스 (트리거 시 점검 · 실제 의존만)

```bash
claude mcp list 2>&1 | grep -i "meta-ads"
test -f "skills/html-report-template/SKILL.md" && echo "리포트 템플릿 ✅"
```

```
  | # | 항목 | 용도 | 상태 |
  |---|---|---|---|
  | ① | Meta Ads MCP (hosted)     | 성과 조회 (읽기 전용)  | {✅ / ❌} |
  | ② | html-report-template 스킬 | 리포트 디자인          | {✅ / ❌} |
  | ③ | Notion 통합               | 리포트 아카이브        | {✅ / ❌} |
  | ④ | Discord webhook           | TOP 3 요약             | {✅ / ❌} |

  ❌ Meta Ads MCP 미등록 시:
  `claude mcp add --transport http meta-ads https://mcp.facebook.com/ads`
   → 첫 호출 시 브라우저 OAuth (✋ 사용자) → 광고 계정 목록 조회로 검증
   → ⚠️ 신규 등록 시 세션 재시작 1회. 샘플 모드는 ① 없이도 진행 가능 (건너뛰기 허용).
```

📐 **스킬? 에이전트?** → 도구(Meta Ads) + 산출(HTML·Notion·Discord)이 있으니 🤖 **에이전트**. 리포트 디자인은 html-report-template **스킬**을 공유한다(6-2~6-4 동일 = DRY). 5요소 골격은 3-1과 같다.

💡 핵심 포인트
  · KPI 5종 = CTR · CPC · CPA · ROAS · 전환율, 각각 계산식과 함께 이해
  · 캠페인 목적(전환/트래픽/도달)에 따라 ROAS 해석이 달라짐 — ROAS 단독 판단 금지
  · 🛡 읽기 전용 : 이 에이전트는 캠페인 상태·예산을 절대 바꾸지 않는다

⏸ 여기까지 이해되셨으면 "2단계" 라고 답해 주세요.

## 2단계 · 실행

### 2-0 · 사전 점검
1단계의 연결 소스 표에서 ❌ 인 항목을 먼저 해결한다.
샘플 모드면 Meta Ads MCP ❌ 라도 그대로 진행 가능.

### 2-1 · 작동 구조
```
  [기간 (기본 최근 7일) + 레벨 (캠페인/세트/광고)]
    → Meta Ads 조회 : 지출·노출·클릭·전환·매출
    → KPI 계산 : CTR · CPC · CPA · ROAS · 전환율
    → 전주 대비 증감 (WoW)
    → ROAS TOP 3 / BOTTOM 3 선별
    → HTML 리포트 (html-report-template 스킬) → outputs/{날짜}/
    → Notion 아카이브 1행 + Discord TOP 3
```
📄 에이전트 정의에서 새로 볼 부분 (`agents/part6-ads/meta-ads-analyzer.md`):
  ① 안전 원칙 명문화 — "읽기 전용 · 상태 변경 금지" 가 프롬프트 레벨에서 선언됨
  ② html-report-template 스킬 재사용 — Part 6 리포트 전부가 같은 디자인 (일관성)
  ③ 산출물 3곳 동시 도달 — 로컬 HTML (감사) + Notion (보관) + Discord (알림)

퀴즈: "이 에이전트에게 '부진 캠페인 꺼줘'라고 하면?"
(답: 거부. 읽기 전용 원칙 + 변경은 사용자 승인 영역)

### 2-2 · 데이터 준비

```
📊 데이터 모드:

  [A] 샘플 데이터 모드 ⭐ 추천 · 촬영 기본
      sample-data/meta-ads-performance.json 로드 (결로아 가상 계정).
      캠페인 3 (챌린지 전환 / 론칭 트래픽 / 인지 리치) + 세트 6 · 7일 + 전주.
      실계정 수치가 영상에 노출되지 않고, ROAS 우수·부진이 보장되어 리포트가 선명함.
      ※ 픽스처는 절대 수정하지 않는다 (재현성 보존).

  [B] 실계정 모드
      본인 Meta 광고 계정 라이브 조회 (읽기 전용 · 상태 변경 절대 없음).
      광고 계정 목록 조회 → 대상 계정 선택 ⏸ → 최근 7일 데이터 존재 확인.
      ⚠️ 촬영 시 계정 지표가 화면에 노출됨을 확인하세요.
```

⏸ A/B 답 받고 2-3 으로.

### 2-3 · 실제 실행
```
▶ 에이전트 워크플로 단계별 실행:

  1. 데이터 로드 (7일 + 전주)
  2. KPI 계산표                          → 캠페인×지표 매트릭스 (시연 하이라이트)
       · 챌린지 전환 : ROAS 3.80 (전주 3.28 → +16%) ↑
       · 론칭 트래픽 : ROAS 1.72 (전주 -15%) ↓
       · 인지 리치   : ROAS 0.43 · 목적이 도달이므로 ROAS 단독 판단 주의 주석
  3. TOP 3 / BOTTOM 3 선별 (세트 레벨)   → 관심사: 스킨케어·성분 세트 4.53 1위
  4. HTML 리포트 생성                     → outputs/{날짜}/meta-roas-{날짜}.html
       → 브라우저로 열어 확인 (표 + 증감 화살표 + 코멘트)
  5. Notion "광고 리포트 아카이브" 1행
  6. Discord TOP 3 embed
```

⏸ 리포트 화면 확인 후 "3단계" 라고 답해 주세요.
