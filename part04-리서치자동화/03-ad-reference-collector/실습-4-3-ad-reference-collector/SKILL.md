# Part 4 / 4-3 · 광고 레퍼런스 수집기 실습 : 소개 → 실행 → 활용

> Meta 광고 라이브러리에서 경쟁 광고를 수집해 **카탈로그화하고 반복 패턴을 추출**하는 에이전트를 가동한다.
> 핵심 학습 목표 : ① 공개 광고 데이터 활용 ② 관련성 필터 ③ 패턴 인사이트 (CTA·소구·포맷).
> ⛔ 자동 실행 금지. 단계마다 사용자 답을 받은 뒤에만 다음으로.

## 진행 흐름 (3 단계)

```
1단계  소개 — 하는 일 + Before/After + 연결 소스 + 핵심 포인트   ⏸
2단계  실행 — 사전 점검 → 작동 구조 → 데이터 준비 → 실제 실행    ⏸
3단계  활용 — 내 업무 적용법 + 산출물 + 다음 클립                ⏸
```

---

## 1단계 · 소개

```
🗂 광고 레퍼런스 수집기를 소개합니다.

이 에이전트가 하는 일 :
  "경쟁사가 돈 내고 검증한 크리에이티브를 매주 자동으로 카탈로그화하고,
   CTA·소구·포맷의 반복 패턴을 뽑아낸다."

⏱ Before vs After
  · Before : 광고 라이브러리 수동 서핑 + 스크린샷 + 시트 정리 = 약 2시간/주
  · After  : 주 1회 자동 수집 30~80건 + 패턴 요약 = 읽는 데 5분/주

🔌 연결 소스 (실제 의존만)
  ┌─ Meta Ads MCP (hosted) ── 광고 라이브러리 검색 (공개 데이터)
  ├─ Google Sheets MCP ────── "광고 레퍼런스 카탈로그" 행 단위 적재
  ├─ Notion ──────────────── "이번 주 인사이트" 페이지 (비정형 분석)
  └─ Discord webhook ──────── 수집 요약 embed

💡 핵심 포인트
  · 출처가 공개 데이터(광고 라이브러리) → 크롤링 부담 없는 합법적 경쟁 분석
  · 적재(Sheets·정형) 와 인사이트(Notion·비정형) 의 분리 저장
  · append-only 설계 → 주간 실행마다 누적되어 시계열 패턴 분석 가능
  · 🔗 이 카탈로그는 Part 5 의 ad-copy-ab(광고 카피 10패턴 생성)가 참조한다
```

📐 **스킬? 에이전트?** → 도구(Meta Ads·Sheets·Notion) + 트리거(주 1회) + 산출(Discord)이 있으니 🤖 **에이전트**. 만드는 법은 3-1의 **에이전트 5요소** 골격 그대로.

⏸ 여기까지 이해되셨으면 "2단계" 라고 답해 주세요. 실제로 가동해 봅니다.

## 2단계 · 실행

### 2-0 사전 점검

```bash
claude mcp list 2>&1 | grep -iE "meta-ads|google-sheets"
test -f "discord-bot/webhook-config.json" && echo "webhook ✅"
```

```
🔧 의존 항목 점검 결과

  | # | 항목 | 용도 | 상태 |
  |---|---|---|---|
  | ① | Meta Ads MCP (hosted)   | 광고 라이브러리 검색 | {✅ / ❌} |
  | ② | Google Sheets MCP       | 카탈로그 적재        | {✅ / ❌} |
  | ③ | Notion 통합 (Claude.ai) | 인사이트 페이지      | {✅ / ❌} |
  | ④ | Discord webhook         | 수집 요약            | {✅ / ❌} |
```

- **① Meta Ads MCP** : `claude mcp add --transport http meta-ads https://mcp.facebook.com/ads` → 첫 호출 시 브라우저 OAuth 동의(Facebook). ⚠️ 신규 등록 시 세션 재시작 1회.
- **② Google Sheets MCP** : Part 2 에서 발급한 OAuth credentials 재사용. 시트 목록 호출로 검증.
- **③ ④ Notion · webhook** : 이전 실습과 동일 절차.

### 모드 선택

```
📊 데이터 모드:
  [A] 샘플 데이터 모드 ⭐ 추천 · 촬영 기본
      K-뷰티 광고 30건 픽스처 (MEDICUBE·ANUA·NUMBUZIN 등 다수 광고주).
      CTA·소구·포맷 패턴이 고의로 군집되어 있어 인사이트가 선명하게 나옴.
  [B] 실데이터 모드
      Meta 광고 라이브러리 실검색 (키워드 6개 · 한국 · 최근 14일). 결과 가변.
```

⏸ A/B 답을 받은 뒤 2-1 로.

### 2-1 작동 구조

```
🔄 에이전트 워크플로
  [검색 키워드 6개 : 토너·모공·21일 챌린지·수분·나이아신아마이드·토너 패드]
    → Meta 광고 라이브러리 검색 (한국 · 최근 14일)
    → 관련성 3점 이상 필터
    → Google Sheets "광고 레퍼런스 카탈로그" 적재 (8컬럼)
    → 반복 패턴 추출 : CTA 분포 / 소구 유형 / 포맷 트렌드
    → Notion 인사이트 페이지 + Discord 수집 요약
```

퀴즈: "왜 카탈로그는 Notion 이 아니라 Sheets 에 쌓을까?" (답: 행 단위 누적·필터·정렬)

⏸ 답을 받고 2-2 로.

### 2-2 데이터 준비

`sample-data/ad-keywords.json` 을 읽어 (⚠️ sample-data 는 수정 금지·픽스처 보존):

1. 검색 키워드 6개 + 필터 (국가 KR · 14일 · 3점 이상) 표로 확인
2. Google Sheets **"광고 레퍼런스 카탈로그"** 생성 (8컬럼: 수집일·광고주·카테고리·헤드라인·본문 발췌·CTA·이미지 URL·관련성 점수) + **"이번 주 인사이트"** 시트 추가
3. 시트 URL 출력

[B] 실데이터 모드면: 본인 업종 키워드로 교체 질문.

⏸ OK 받고 2-3 으로.

### 2-3 실제 실행

```
▶ 워크플로 단계별 실행:

  1. 키워드 6개 확인
  2. 광고 수집
       [A] 샘플 : ad-library-results.json 30건 로드 (collected_date 2026-06-10)
       [B] 실데이터 : Meta 광고 라이브러리 검색 (키워드당 5~15건)
  3. 관련성 필터        → 30건 전건 3점 이상 통과 (점수 분포 표)
  4. Google Sheets 적재 → 30행 (광고주 다수)
  5. 패턴 인사이트 추출 (하이라이트) → 4건:
       · CTA  : 직접 구매형 12건 최다 + 챌린지·체험단형 5건 급증
       · 소구  : 수치 증명형 (임상 23%·리뷰 12만·판매 1천만) 반복
       · 포맷  : 토너 패드 신제품 3개 브랜드 동시 등장 → 패드 경쟁 심화
       · 벤치마크 : ANUA 7일 챌린지 참여 3만 명 → 결로아 21일 챌린지 직접 비교
  6. Notion 인사이트 페이지 생성 + Discord 수집 요약 embed
```

⏸ 각 단계 결과 확인 후 다음 단계. 6단계까지 마치면 "3단계" 로.
