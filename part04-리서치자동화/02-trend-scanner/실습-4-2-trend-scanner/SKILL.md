# Part 4 / 4-2 · 트렌드 스캐너 에이전트 실습 : 소개부터 급상승 TOP 3 까지

> **시드 키워드 1개 → 연관어 확장 → 3채널 검색 → 출처 추적 채점 → 인사이트 → 통합 시장조사 자료**.
> 핵심 학습 목표 : ① 키워드 입력형 분석 ② 검색 소스·도구 외부화 + 출처(네이버/구글) 추적
>   ③ 인사이트 합성(패턴→시사점→액션) ④ 파이프라인 체이닝 (4-2 → 3-1).
> ⛔ 자동 실행 금지. 각 단계마다 사용자 답을 받은 뒤에만 다음으로.

## 진행 흐름 (3 단계)

```
1단계  소개 — 하는 일 + 연결 소스 + 핵심 포인트       ⏸ 답 대기
2단계  실행 — 사전 점검 → 작동 구조 → 데이터 준비 → 실제 실행   ⏸ 단계별 게이트
3단계  활용 — 내 업무 적용법 + 산출물 + 다음 클립      ⏸ 선택
```

각 단계 끝의 ⏸ 에서 멈추고, 사용자 답을 받은 뒤에만 다음 단계로 넘어간다.

### 2-2 · 트렌드 스캔 설정 빌드 (한 칸씩)

> **이 단계가 4-2의 핵심 학습이다.** "내가 설정을 세우면 → 에이전트가 그 기준대로 찾고 거른다"
> 를 학습자가 직접 경험하게, 설정 7칸을 **한 칸씩** 세운다. 각 칸은 샘플 기본값을 제시하고
> `[그대로 ⏎]` 또는 `[내 걸로 교체]` 중 선택받는다. 픽스처를 통째로 시딩하지 않는다.

먼저 진행 속도를 고른다:

```
⚙️ 설정을 어떻게 세울까요?

  [빠르게] 7칸 샘플 기본값(결로아 K-뷰티)을 한 번에 적용 → 바로 2-3   · 촬영·시연용
  [하나씩] ⭐ 7칸을 차례로 세우며 각 칸의 의미를 체득 → 내 브랜드 적용
```

**[하나씩] 선택 시** — 아래를 한 칸씩 질문한다. 각 칸: 샘플 기본값 제시 → ⏸ 답 받고 다음 칸.
(샘플 기본값 출처: `sample-data/trend-scan-config.md` · 읽기 전용 · 수정 금지)

```
① 카테고리 / 브랜드   기본: K-뷰티 D2C (토너·세럼) · 결로아
     → 무엇의 트렌드인가. 관련성 판단의 '맥락'이 된다.

② 검색 소스 & 도구    기본: 아래 매핑 표 (어디서 / 무엇으로 찾을지)
     → '무엇을' 만큼 중요. 소스·도구·쿼리 방식을 설정으로 외부화한다.

③ 상시 추적 키워드     기본: 모공·유분·수분·챌린지·21일·토너 패드·PHA·저자극
     → 실행 시 시드 키워드를 입력하면 그걸 우선 분석.
       시드를 안 주면(정기 스캔) 이 목록으로 검색·채점. 시드의 관련도 판정 기준도 겸함.

④ 추적 해시태그       기본: #모공케어 #토너추천 #21일챌린지 #케이뷰티 #스킨케어루틴
     → SNS 신규성 신호.

⑤ 제외어              기본: 메이크업·아이섀도·립·향수
     → 자동 드롭 = 노이즈 차단. (예: '챌린지' 키워드가 운동챌린지를 잡는 오탐을 막음)

⑥ 채점·적재 기준      기본: 5점=키워드2개↑+직접언급 / 3~4점=1개 / 0~2점=드롭 · 적재는 3점↑

⑦ 급상승 임계         기본: 전주 대비 1.5배↑ 또는 신규 등장 → TOP 3 만 Discord
```

**② 검색 소스 & 도구 — 상세 매핑** (각 행 ON/OFF·쿼리 수 조절 가능)

```
| 소스 채널        | 도구 (MCP)               | 쿼리 / 스코프                  | 결과 출처 표기        | 기본   |
|-----------------|-------------------------|-------------------------------|---------------------|-------|
| 네이버 (블로그·카페) | Firecrawl (search/scrape) | "토너패드 site:naver.com" 검색·URL 스크랩 | "네이버" + 매체 | ✅ ON (키 필요) |
| 구글 / 일반 검색  | WebSearch               | 도메인 제한 없음               | "구글" + 매체 도메인   | ✅ ON |
| 뉴스             | WebSearch               | 키워드 + 연도/계절             | "뉴스" + 매체         | ✅ ON |
| SNS 해시태그      | WebSearch               | #해시태그                      | "SNS" + 플랫폼        | ✅ ON |
| 특정 매체 심층     | Firecrawl (search/scrape)| 지정 도메인·URL                | 매체 도메인           | ⬜ 선택 |
| 고정 URL 모니터   | WebFetch                | 설정한 URL 목록                | 해당 URL             | ⬜ 선택 |

📌 출처 표기 규칙: **모든 적재 행은 '출처' = (검색 채널) + (실제 매체 도메인)** 을 기록한다.
   예) "네이버 · blog.naver.com" · "구글 · hwahae.co.kr" · "SNS · threads.com"
   → Notion DB '출처' 속성과 Discord embed 에 함께 표기 → 결과가 어디서 왔는지 추적 가능.
   (매체는 결과 URL 도메인에서 읽는다.)

⚠️ 정직 고지 (실측 확인):
   · **WebSearch 크롤러는 naver.com 을 못 긁는다** (allowed_domains=naver.com → API 400).
     따라서 네이버는 **Firecrawl 경로 필수** (`.env` FIRECRAWL 키 필요).
   · Firecrawl 미설치/미승인 시 → 네이버 레인 자동 비활성, **구글 일반 검색 결과 중
     URL 이 naver.com 인 건만 출처 "네이버"로 라벨**하는 무료 폴백으로 동작.
   · 네이버 데이터랩·X·인스타 공식 API 는 전용 MCP 부재 → WebSearch 우회 또는 제외.
조절 knob : (a) 행별 ON/OFF (b) 쿼리 개수(검색 비용↔커버리지) (c) blocked 도메인.
```

7칸이 끝나면:
1. 누적된 설정을 표로 보여주고 확인받는다.
2. Notion 페이지 **"트렌드 스캔 설정"** 에 기록 (있으면 **업데이트** · 없으면 생성).
3. Notion DB **"주간 트렌드"** 확인 (3-1 실습에서 만들었으면 재사용 · 없으면 생성).

🔑 핵심 멘트: "방금 ②에서 어디서·무엇으로 찾을지, ③·⑤에서 키워드·제외어를 직접 정했죠?
2-3에서 에이전트가 **정확히 이 설정대로** 그 소스를·그 도구로 찾고 거르는 걸 보게 됩니다.
코드(agent.md)는 한 줄도 안 건드렸습니다."

**[빠르게] 선택 시** — 7칸 샘플 기본값을 한 번에 적용·기록하고 바로 검색 모드로.

그 다음 **검색 모드**를 고른다:

```
📊 검색 모드:
  [샘플]   픽스처 8건으로 채점을 결정적으로 시연 (촬영 기본)
  [실데이터] WebSearch 실검색 — 결과 가변 · 3~5분. 설정에 맞춰 실제로 찾아옴.
```

⏸ 설정 기록 + 검색 모드 확정 후 2-3 으로.

### 2-3 · 실제 실행 (단계별 게이트)

```
▶ 워크플로 단계별 실행:

  0. 시드 키워드 입력                 → "분석할 키워드를 입력하세요" (예: 토너패드)
       · 입력 없으면 설정의 '상시 추적 키워드' 전체로 정기 스캔
  1. Notion 설정 페이지 로드          → 제외어·채점기준·검색 소스/도구 매핑 표
  2. 연관 키워드 확장                 → 시드 → 관련어 5~8개 자동 생성·제시
       · 예: 토너패드 → 모공패드·각질패드·진정패드·물광패드·올영토너패드…
  3. 검색 수행 (설정의 소스·도구·스코프대로 · 확장 키워드로)
       [샘플]   픽스처 8건을 검색 결과로 사용 (각 건에 출처 채널 부여)
       [실데이터] 소스 레인별 실행
                 (네이버=Firecrawl site:naver.com · 구글=WebSearch 제한없음 · 뉴스 · 해시태그)
                 ※ WebSearch 는 naver.com 차단 → 네이버는 반드시 Firecrawl, 없으면 폴백
  4. 관련성 채점                      → 채점표 (열: 제목·**출처**·매칭·점수·판정)
       · 점수 = 시드 키워드와의 관련도 · 출처 = 검색 채널 + 매체 도메인
       · 5점·3~4점 적재 / 제외어 = 드롭 사유 표시
  5. 급상승 판정                      → 배율 또는 신규/매체빈도 신호 (출처 분포도 함께)
  6. 인사이트 합성 ★                  → 패턴(3~5) → 시사점 → 액션 3~5 표
       · (a) 점수·출처·키워드로 패턴 클러스터링
       · (b) 각 패턴의 마케팅 함의 (왜 중요한가)
       · (c) 실행 액션 (콘텐츠/포지셔닝/채널/성분) + 근거 데이터 연결
       · 출처 분포를 퍼널로 해석 (뉴스=인지 / 비교랭킹=고려 / 네이버 후기=전환)
  7. Notion "주간 트렌드" DB 적재       → 통과분 전량 적재 (행마다 '출처' 기록 · 원천 데이터)
  8. 통합 시장조사 자료 1페이지 생성 ★  → 인사이트 + 주간 트렌드를 한 페이지로
       · 5섹션: ①개요 ②핵심 인사이트(패턴→시사점→액션) ③수집 트렌드 표(출처·점수)
                ④출처 분포·급상승 ⑤다음 액션(3-1 뉴스레터 체이닝)
       · 이것이 사용자에게 전달하는 최종 산출물
  9. Discord #marketing-research 발송  → 급상승 TOP 3 (출처 표기) + 💡추천 액션 필드
  10. 정리 ★                          → 중복 페이지 제거(통합본 리다이렉트 스텁) · 오타·카테고리 교정
```

⏸ 각 단계 결과 확인 후 다음 단계로. 10 까지 끝나면 3단계로.
