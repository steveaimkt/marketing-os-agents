# Part 4 / 4-1 · 경쟁사 모니터링 에이전트 실습 : 소개 → 실행 → 활용

> 경쟁사 사이트를 매일 스크랩해 **어제와 달라진 점만** 분류·보고하는 에이전트를 가동한다.
> ⛔ 자동 실행 금지. 각 단계마다 사용자 답을 받은 뒤에만 다음으로.

## 진행 흐름 (3 단계)

```
1단계 · 소개   하는 일 + 연결된 소스 안내                    ⏸ 확인 후 진행
2단계 · 실행   사전 점검 → 작동 구조 → 데이터 준비 → 실제 실행  ⏸ 단계별 확인
3단계 · 활용   내 업무 적용법 + 산출물 + 다음 클립             ⏸ 선택
```

---

## 1단계 · 소개

**하는 일** : "매일 전체를 보는 게 아니라, 달라진 것만 본다 (스냅샷 diff)."

⏱ Before vs After
- Before : 경쟁사 3사 사이트·인스타 순회 20분/일 × 주 5회 = 약 1시간 40분/주, 그나마 놓침
- After  : 매일 자동 스크랩 → 변경점만 디스코드로 도착 = 읽는 데 1분/일

🔌 연결된 소스 (실제 의존만)
```
competitor-monitor 에이전트
├─ Firecrawl MCP   경쟁사 사이트 스크랩 (본문 텍스트 추출)
├─ Notion          경쟁사 목록 + 스냅샷 아카이브
└─ Discord webhook 변경 리포트 발송
```

핵심 포인트
- ① Firecrawl 스크랩  ② 스냅샷 diff 패턴  ③ 변경 5분류 (신제품/가격/카피/이벤트/비주얼)
- 변경이 없는 날은 디스코드에 **아무것도 보내지 않는다** (알림 피로 방지)

📐 **스킬? 에이전트?** → 도구(Firecrawl·Notion) + 트리거(매일) + 산출(Discord)이 다 있으니 🤖 **에이전트**.
   만드는 법은 3-1에서 배운 **에이전트 5요소**(frontmatter·tools·trigger·workflow·outputs) 골격 그대로.

⏸ "소개 확인했으면 2단계로 갈까요?" 답 받고 진행.

## 2단계 · 실행

### 2-0. 사전 점검 (조용히)

```bash
claude mcp list 2>&1 | grep -i "firecrawl"
grep -q "FIRECRAWL_API_KEY" .env 2>/dev/null && echo "key ✅"
test -f "discord-bot/webhook-config.json" && echo "webhook ✅"
```

| 항목 | 용도 | 해결 |
|---|---|---|
| Notion 통합 (Claude.ai) | 경쟁사 목록 + 스냅샷 아카이브 | 1분 (1클릭) |
| Firecrawl MCP (실호출) | 경쟁사 사이트 스크랩 | 2분 (API key) |
| Discord webhook | 변경 리포트 발송 | URL 등록 |

Firecrawl 은 키 존재만으로 부족 → **단일 페이지 스크랩 1건으로 실호출 검증** (무료 월 500 크레딧, 잔여 표시).
신규 설치 시 세션 재시작 1회 후 재호출. ❌ 항목만 골라 지금 해결. ⏸ 점검 OK 받고 다음으로.

### 2-1. 작동 구조 보여주기

```
[Notion 경쟁사 목록 : 3~10사]
  → Firecrawl 로 각 사이트 스크랩 (본문 텍스트 추출)
  → 어제 스냅샷과 텍스트 diff
  → 변경 자동 분류 : 신제품 / 가격 / 카피 / 이벤트 / 비주얼 (5분류)
  → "왜 중요한가" 인사이트 3~5건 추출
  → Notion 스냅샷 아카이브 저장 + Discord 변경 리포트 (변경 없으면 조용히 종료)
```

에이전트 정의 열람 (학습 핵심):
```bash
open "agents/part4-research/competitor-monitor.md"
```
- ① trigger : schedule "매일 07:30" (3-1 의 주간과 다른 일간 주기)
- ② diff 로직 : 전체 저장이 아니라 "변경만 보고" 하는 설계
- ③ 에러 처리 : 사이트 403 차단 시 대기 후 재시도 / robots.txt 존중

퀴즈: "변경이 하나도 없는 날, 이 에이전트는 디스코드에 뭐라고 보낼까?" (답: 안 보냄)
⏸ 답 받고 데이터 준비로.

### 2-2. 데이터 준비 — 모드 선택

```
📊 데이터 모드:
  [A] 샘플 데이터 모드 ⭐ 추천 · 촬영 기본
      경쟁사 3사 (MEDICUBE·TORRIDEN·NUMBUZIN) + 전일 스냅샷 픽스처 선시딩.
      오늘 스냅샷도 픽스처 시나리오 사용 → diff 결정적 재현 (Firecrawl 크레딧 0 소모).
  [B] 실데이터 모드
      경쟁사 3사를 Firecrawl 로 실제 스크랩 (크레딧 ~6 소모 · diff 결과 가변).
      전일 스냅샷은 픽스처 사용 (첫 실행이므로).
```

사용 픽스처 (수정 금지):
- `sample-data/competitors.json` → Notion DB **"경쟁사 목록"** 3사 시딩
  (브랜드명·도메인·인스타·광고 라이브러리 URL·카테고리 / 5분류: 신제품·가격·카피·이벤트·비주얼)
- `sample-data/snapshot-yesterday.md` → Notion DB **"경쟁사 스냅샷 아카이브"** 에 전일 스냅샷 3건 선시딩 (날짜: 어제)
  - 어제값: MEDICUBE 제로 모공 패드 2.0 / 29,000원 · TORRIDEN 다이브인 세럼 50ml / 22,000원 · NUMBUZIN 3번 세럼 / 24,900원
  - 이 선시딩 덕분에 첫 실행에서도 비교 대상이 존재

[B] 실데이터 모드면: 본인 경쟁사 3사로 교체 입력 받기 (도메인 필수). ⏸ 시딩 결과 OK 받고 실행으로.

### 2-3. 실제 실행

```
▶ 에이전트 워크플로 단계별 실행:
  1. Notion 경쟁사 목록 로드                      → 3사 표
  2. 오늘 스냅샷 확보
       [A] 샘플 : snapshot-yesterday.md 의 "오늘 변화 시나리오" 사용
       [B] 실데이터 : Firecrawl 스크랩 3건 (크레딧 차감 표시)
  3. 어제 스냅샷과 diff                           → 변경 검출:
       · MEDICUBE  : 이벤트+가격 ("여름 모공 빅세일 최대 50%")
       · TORRIDEN  : 변경 없음 (보고 생략됨을 시연)
       · NUMBUZIN  : 신제품 (5번 토너 패드 베스트 1위 / 26,900원)
  4. 인사이트 3건 추출                            → "결로아 21일 챌린지 시기와 겹침" 등
  5. Notion 아카이브에 오늘 스냅샷 + 변경 분류 저장
  6. Discord 변경 리포트 embed 발송
```

⏸ 각 단계 결과 확인 후 다음 단계.

## 3단계 · 활용 — 지금, 내 업무로

### 적용 게이트 (능동 질문 ⏸)

```
🚀 그럼, 지금부터 내 업무에 적용해볼까요?

   샘플 경쟁사 자리에 당신 경쟁사만 넣으면 똑같이 작동합니다.
   ① 어떤 브랜드/업종인가요?           (예: 비건 스킨케어)
   ② 감시할 경쟁사 2~3곳은?            (브랜드명 또는 사이트 주소)
   ③ 무엇의 변화가 제일 궁금한가요?    (가격 · 신제품 · 카피 · 이벤트 중)
```

→ 답을 받으면 : 그 경쟁사 대상으로 변경 감시 1회 즉석 실행 → 5분류 + 인사이트 3건.
→ "건너뛰기" 시 아래 정착 가이드만.

### 오래 쓰는 법 (적용 후 정착)
- A. 본인 경쟁사 3사로 `competitors.json` 교체 후 매일 변경점만 1분에 확인
- B. 신제품/가격 변경이 잡히면 → 5-2 광고 카피·5-3 상세페이지에 즉시 대응 메시지
- C. "이벤트" 분류가 우리 캠페인 시기와 겹치면 → 차별화 메시지 우선순위로 격상
- D. 실전 전환 : 매일 07:30 cron 등록 (Part 10) → 출근 전 변경 리포트 도착
