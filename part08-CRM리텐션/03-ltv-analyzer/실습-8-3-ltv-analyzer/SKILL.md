# Part 8 / 8-3 · LTV 분석 에이전트 실습 : 소개 → 실행 → 활용

> 통합 고객 마스터 DB 를 **4세그먼트(VIP·Growing·At-Risk·Dormant)로 나눠 재구매·이탈 패턴과 액션**을 뽑는다.
> 핵심 학습 목표 : ① 4세그먼트 정의·판정 ② 인원 vs 매출 비중의 비대칭 ③ 이탈 신호 → 액션 연결.
> ⛔ 자동 실행 금지. 단계마다 사용자 답을 받은 뒤에만 다음으로.

```

  └ 2-4 (선택) 심화 확장 — GA4 → 액션 플랜 + 카피 → 예산 재배분      ⏸ "확장" 선택 시

## 2단계 · 실행

### 2-0 · 사전 점검
1단계 표에서 ❌ 를 확인한다. 샘플 모드면 ②③④ ❌ 라도 그대로 진행 가능.
① 은 8-1 을 먼저 돌렸으면 그 마스터를, 아니면 sample-data 30명 픽스처를 쓴다.

### 2-1 · 작동 구조
```
  [고객 마스터 DB (8-1 산출 · 12 컬럼)]
    → 세그먼트 판정 (기준: 마지막 주문일 + 누적 매출)
        VIP / Growing / At-Risk / Dormant  (+ 리드=미구매)
    → 세그먼트별 집계 : 인원·인원비중 / 매출·매출비중 / 재구매 간격(median)
    → 이탈 신호 : (last_contact - last_order) 분포 · 광고/CS 재접촉 여부
    → 추천 액션 5~10개 (세그먼트별 우선순위)
    → Notion 월간 LTV 인사이트 + Sheets 액션 탭 + Discord embed
```
📄 에이전트 정의에서 새로 볼 부분 (`agents/part8-crm/ltv-analyzer.md`):
  ① 4세그먼트 정의·추천 액션이 표로 명문화 (VIP 로열티 / At-Risk 윈백 …)
  ② 입력이 customer-data-unifier 결과 — 8-1 없이는 8-3 이 성립 안 함(파이프라인)
  ③ 이탈 직전 신호 = last_contact_date 와 last_order_date 의 간극 분포

퀴즈: "마지막 주문 85일 전인데 광고는 24일 전에 클릭한 고객, 세그먼트와 신호는?"
(답: At-Risk(1회+60~120일 무주문) · '관심은 있는데 안 사는' 강한 이탈 직전 신호 → 윈백 1순위)

```

  1. 마스터 DB 로드 (30명 · 총매출 2,650,000)
  2. 세그먼트 판정표                      → (시연 하이라이트)
       · VIP 4명 (13.3%) · 매출 1,055,000 (39.8%) · 재구매 median ~30일
       · Growing 10명 (33.3%) · 매출 996,000 (37.6%)
       · At-Risk 8명 (26.7%) · 매출 333,000 (12.6%) · 60~120일 무주문
       · Dormant 8명 (26.7%) · 매출 266,000 (10.0%) · 120일+
  3. 이탈/기회 신호                        → 개별 하이라이트
       · At-Risk C015 최민준 : 마지막 주문 85일 전, 광고 클릭 24일 전(미구매) = 강한 이탈 신호

  6. Discord embed → #marketing-alerts    → 세그먼트별 인원/매출비중/우선 액션

### 2-4 · (선택) 심화 확장 — LTV → GA4 → 액션 플랜 → 카피 → 예산

> "누가"(세그먼트)에 GA4 행동 데이터를 얹어 **"어디서·왜"** 까지 채우고, 세그먼트별 **실제 발송 문구**와 **광고 예산 재배분**까지 뽑는다. "확장" 을 고르면 진행.
> 파이프라인 : `8-1 마스터 → 8-3 LTV 4세그먼트 → GA4 세그먼트×유입 → 액션 플랜 4종+카피 → 예산 재배분`

**(1) GA4 샘플 (결로아 연장 · 2026-06 · 결정적 재현 · 전부 가상)**
전체 : 세션 4,180 · 사용자 3,090 · 구매전환율 2.4% · 장바구니 이탈률 68%
Exit 페이지 : ①/cart 41% ②/product/신상(로딩 3.8s) 22% · Landing : ①/event/first15 ②/product/best

```
  | 세그먼트 | 주 유입 | 2순위 | 전환율 | 장바구니이탈 | GA4 읽는 법 |
  |---|---|---|---|---|---|
  | VIP     | 직접 45%      | 이메일 30% | 6.1% | 40% | 브랜드 인지 완료 → 푸시/딥링크가 먹힘 |
  | Growing | 검색(오가닉) 40% | 이메일 25% | 3.2% | 61% | SEO·콘텐츠 유입 → 번들 랜딩 최적화 |
  | At-Risk | 광고(유료) 48%  | 검색 30%   | 1.1% | 78% | 광고로 사고 이탈 → 광고 리타겟+알림톡 이중 |
  | Dormant | SNS 32%       | 광고 26%   | 0.6% | 81% | 인스타 유입 → SNS 리타겟만(저비용) |
```

**(2) GA4 → 액션 플랜 + 실제 카피 4종**
```
  · At-Risk : 광고 리타겟(메타/네이버)+알림톡 병행 · 랜딩=원클릭 재주문(장바구니 78% 이탈 우회)
      [알림톡] "○○님, 담아두신 상품 아직 기억하시죠? 15% 쿠폰 먼저 넣어뒀어요(~D+7). 👉 이어서 보기"
  · VIP : 앱푸시 딥링크로 신상 바로 진입(로딩 3.8s 우회)+속도개선 티켓
      [이메일] 제목 "○○님께 먼저 여는 문 — 신상, 공개 24시간 전" / 본문=충성 인정+우선 입장 링크
  · Growing : 정기배송 토글을 장바구니 페이지에 노출(61% 이탈 지점에서 잡기)+SEO 강화
      [이메일] 제목 "3번째 주문 축하 — 이제 정기배송으로 편하게(첫 회 20%)"
  · Dormant : 인스타 리타겟만, 예산은 재접촉 신호자(C023·C030)에 한정
      [리타겟] "다시 볼 줄 알았어요 👀 그때 그 상품, 컴백 기념 -20% 이번 주만."
```

**(3) 예산 재배분** : At-Risk 리타겟 40% · Growing SEO/정기배송 35% · Dormant 신호자 리타겟 25% · **VIP 무광고(오가닉 유지)**.
→ 광고비는 *살아있는 관심(At-Risk)* 에 집중, VIP 는 광고 없이 지킴 = 이탈 방지 최고 ROI.

**(4) 산출** : 기존 Sheets 에 `GA4_세그먼트유입` 탭 추가 + Notion LTV 페이지에 "GA4·액션 플랜·카피" 섹션 append.
