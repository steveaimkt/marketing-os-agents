# Part 8 / 8-2 · 고객 CS 응대 에이전트 실습 : 가이드 구축 → 문의·답변 → 디스코드

> **CS 문의 엑셀을 넣으면, 우리가 늘 답하던 방식으로 답변을 정리해 디스코드로 보낸다.**
> 핵심 : 사실은 **정책 위키**가, 말투는 **답변 가이드**가 결정하는 2축 구조.
> ⛔ 자동 실행 금지. 단계마다 사용자 답을 받은 뒤에만 다음으로.

## 진행 흐름 (3 단계)

```
1단계  CS 가이드 구축      — 우리 말투(호칭·인사·마무리·금기) 확정      ⏸ 답 대기
2단계  문의 추가 후 답변    — 엑셀 문의 → 분류 + 우리말투 답변 초안        ⏸ 답 대기
3단계  디스코드로 결과 확인 — 파이프라인 실행 → #marketing-approvals 발송  ⏸ 확인
```

📂 사용하는 파일 (sample-data/ · automation/scripts/)

```bash
ls sample-data/ | grep -E "cs-answer-style|cs-inbox|cs-wiki|answers"
#   cs-answer-style.md   ← 답변 가이드 (HOW · 말투)      [1단계에서 다룸]
#   cs-wiki.md           ← 정책 위키   (WHAT · 사실)
#   cs-inbox.xlsx        ← CS 문의 입력 엑셀              [2단계에서 다룸]
#   answers.json         ← 에이전트 산출(분류+답변)       [2단계 결과]
ls ../../../../automation/scripts/cs-excel-to-discord.py   # 발송 파이프라인 [3단계]
```

---

## 1단계 · CS 가이드 구축

```
🗂 먼저 "우리가 답하는 방식"을 정합니다. = 답변 가이드(cs-answer-style.md)

  왜 필요한가?
   · 정책 위키(cs-wiki.md)는 "무엇을(배송 1~2일·3만원 무료)"만 안다.
   · "어떻게(안녕하세요 고객님 😊 … 편하게 문의해 주세요 :))" 는 이 가이드가 정한다.
   · 이 파일만 바꾸면 답변 말투 전체가 바뀐다.
```

📋 가이드가 정하는 4가지 (현재 샘플 = 결로아)

```
  ① 호칭·인사말   : "고객님" · "안녕하세요, 결로아입니다 😊"
  ② 어조·문장     : 정중한 존댓말 · 핵심 먼저 · 2~4문장 · 이모지 0~1
  ③ 분류별 마무리 : 🟢 "편하게 문의해 주세요 :)" · 🟠 "불편을 드려 죄송합니다"(1회)
  ④ 금기          : 과잉 사과 · 효능 단정 · 의학 진단 · 정책 외 약속
```

🔧 지금 할 선택 (⏸)

```
  [A] 샘플 가이드 그대로 사용 ⭐ 추천 · 촬영 기본
      → 결로아 말투 + 대표 답변 예시 7건을 그대로 활용.

  [B] 내 브랜드 말투로 수정
      → 호칭 / 인사말 / 마무리 문구 / 금기 중 바꿀 것을 알려주면
         cs-answer-style.md 를 즉시 수정해 반영.
```

⏸ **[A] 그대로** 또는 **[B] 수정** 을 답해 주세요. (수정 시 바꿀 항목도 함께)

## 2단계 · CS 문의 추가 후 답변 보기

### 2-1 · 입력 엑셀 형식
```
  cs-inbox.xlsx  컬럼 : 문의ID | 접수일 | 채널 | 발신 | 문의내용
  (샘플 7건 = 🟢×2 🟡×2 🟠×2 🔴×1 → 4분류·승인게이트가 한 번에 재현)
```

### 2-2 · 문의 추가 (⏸ 선택)
```
  [A] 샘플 7건 그대로 ⭐ 추천
  [B] 내 문의 추가   → "채널 | 문의내용" 을 알려주면 엑셀에 행 추가 후 함께 처리
```

### 2-3 · 분류 + 우리말투 답변 (에이전트 실행)
```
▶ 엑셀 행마다:
  1. 분류 + 신뢰도 (🟢/🟡/🟠/🔴)
  2. 정책 위키 검색  → 사실 근거 [§x]
  3. 답변 가이드 검색 → 유사 예시 2~3건 참조 [예시 #n]
  4. 초안 = 사실(위키) + 말투(가이드) 결합
  5. 스타일 일치 체크 → 호칭·마무리·금기 위반 시 재작성
  6. 🔴 클레임 → ⛔ 초안 만들지 않음 · CS팀 즉시 알림
```
→ 결과를 분류별 표(원본·초안·[§x]·[예시#n])로 보여준 뒤 answers.json 저장.

퀴즈: "🔴 클레임에 바로 사과 답장을 보내도 될까?"
(답: 안 됨. 🔴 는 답변 안 함 + CS팀 즉시 알림. 법적 판단은 사람.)

⏸ 답변 표 확인 후 "3단계" 라고 답해 주세요.

## 3단계 · 디스코드로 결과 확인

```
▶ 파이프라인 실행 (기계 부분):
   python3 automation/scripts/cs-excel-to-discord.py \
       sample-data/cs-inbox.xlsx sample-data/answers.json
   → 엑셀 읽기 + embed 조립 + 위험순 정렬 + 발송

📤 #marketing-approvals 에 뜨는 것:
   · 헤더  : "CS 문의 N건 정리 완료 · 분류 🔴1·🟠2·🟡2·🟢2 · 승인 대기"
   · 🔴 카드 : "답변 보류 · CS팀 즉시 알림" (초안 없음)
   · 🟠 카드 : 사과 톤 답변 + "@CS팀 멘션"
   · 🟡 카드 : 답변 + "사람 검토 후 발송"
   · 🟢 카드 : 답변 + "✅ 눌러야 발송" (자동발송도 승인 필수)
   · 각 카드 근거 : [위키 §x] + [예시 #n] 이중 표기

🛡 안전 확인 : 정책 외 약속 0 · 의학 진단 0 · 출처 누락 0 · 🔴 답변 0
```

### 3-2 · 노션 응대 로그 미러 (디스코드로 보낸 내용 = 노션에도 저장)
```
▶ 발송한 답변을 노션 "CS 응대 로그" 페이지/DB에 그대로 저장:
   · 각 건 = 원문 + 답변 전문 + 근거([§x]·[예시#n]) + 처리(🟢🟡🟠🔴)
   · 목적 : 감사 추적 · 재사용 · ✏️ 수정분을 가이드 보강 근거로 활용
   ※ 노션 저장은 Notion MCP 로 에이전트가 수행 (파이썬 스크립트는 디스코드만 담당)
```
→ 3곳 도착 완성 : 로컬(answers.json) · 디스코드(승인 embed) · 노션(응대 로그)

⏸ Discord 카드 + 노션 로그 확인 → 아래 활용/다음으로.
