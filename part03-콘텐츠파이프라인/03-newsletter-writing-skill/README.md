# 클립 1-3. 콘텐츠 작성 스킬 구축: 뉴스레터 작성 스킬 만들기

## 한 줄 요약
**여러 에이전트가 공통으로 호출하는 글쓰기 절차서**를 만든다. 한 번 만들고 영원히 재사용.

## 스킬 vs 에이전트 vs 명령

| | 에이전트 | 스킬 | 명령 |
|---|---|---|---|
| **목적** | 워크플로 전체 | 작업 패턴 | 단일 호출 |
| **재사용** | 큰 단위 | 작은 단위 | 1회성 |
| **위치** | `agents/` | `skills/` | `commands/` |
| **호출 방식** | `claude --agent {name}` | Claude가 자동 또는 명시 호출 | `claude /command` |

→ **뉴스레터 작성**은 패턴이라 **스킬**

## 워크플로를 스킬로 만드는 5단계

> 이 클립의 본론. "매주 손으로 하던 워크플로"를 **SKILL.md 한 장**으로 박제하는 법.

| 단계 | 무엇을 | 한 줄 |
|---|---|---|
| **① 반복 신호 포착** | 스킬 후보 판별 | 같은 작업 3번 = 스킬감. *"다음 주에도 똑같이 또 하나?"* → Yes면 스킬 |
| **② 변하는 것 / 안 변하는 것 가르기** | 설계의 핵심 | 매주 바뀜(자료·톤·CTA) → **입력** · 매주 같음(원칙·절차) → **스킬 본문** |
| **③ SKILL.md 4요소 작성** | 빈 파일 → 위에서 아래로 | frontmatter(특히 description) → 입력 스키마 → 출력 스키마 → 검증 체크리스트 |
| **④ 암묵지를 글로** | 품질 고정 | "난 원래 이렇게 써"(제목 5패턴·금기어·이모지)를 **글쓰기 원칙**으로 명문화 |
| **⑤ 호출 테스트 → 에이전트 결합** | DRY 증명 | 직접 호출 1회 → email-newsletter가 `Skill(newsletter-writing)`로 같은 스킬 호출 |

→ **②가 전부다**: 바뀌는 건 입력으로 빼고, 안 바뀌는 건 스킬에 박는다.

## 학습 포인트 (SKILL.md 4요소)
1. **YAML frontmatter** — name + description (Claude가 스킬을 선택할 때 description을 봄)
2. **입력 스키마** — 어떤 파라미터를 받는지 명확히 (= ②의 '바뀌는 것')
3. **출력 스키마** — 무엇을 돌려주는지 (다른 에이전트가 받기 좋게 '모양' 고정)
4. **검증 체크리스트** — 스킬 종료 전 자체 점검 (금기어 0 · 분량 · CTA 1개)
