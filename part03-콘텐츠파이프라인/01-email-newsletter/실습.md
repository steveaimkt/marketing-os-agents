# 클립 1-1 실습 — email-newsletter 에이전트

## Step 1. 에이전트 정의 파일 만들기
이미 [`agents/part3-content/email-newsletter.md`](../../../agents/part3-content/email-newsletter.md)에 작성되어 있다. 한 번 열어 구조를 확인:

```bash
claude "agents/part3-content/email-newsletter.md를 읽고 워크플로를 3줄로 요약"
```

## Step 2. 노션 DB 준비
```bash
claude "$(cat <<'EOF'
Notion에 새 DB 두 개 만들어줘:

1) "주간 트렌드"
   - 제목 (Title), 카테고리 (Select: 뷰티/광고/SNS/리뷰), URL, 요약 (Text), 수집일 (Date)

2) "뉴스레터 아카이브"
   - 제목 (Title), 발송일 (Date), 주차 (Text), 구독자수 (Number),
     오픈률 (Number, %), 노션 미리보기 URL

생성 후 두 DB의 ID를 알려줘.
EOF
)"
```

## Step 3. 첫 테스트 발송 (본인 1명만)
```bash
# 트렌드 데이터를 수동으로 3개 입력한 뒤
claude --agent email-newsletter "$(cat <<'EOF'
테스트 모드:
  - 발송 대상: 본인(me@example.com) 1명만
  - 소스: 노션 "주간 트렌드" DB 최근 3개 행
  - 승인 단계: 건너뛰지 말 것

먼저 디스코드 #marketing-approvals 채널에 미리보기 발송.
사용자 reaction ✅ 받으면 발송.
EOF
)"
```

## Step 4. 디스코드 reaction으로 발송 승인
미리보기 메시지에 ✅ reaction → 본문이 본인 메일함에 도착
❌ reaction → 발송 안 됨, 노션 아카이브에 "취소" 표시
