# Part 3 / 3-1 · 이메일 자동 발송 에이전트 실습

> 매주 월요일, Claude 가 지난 주 트렌드를 모아 뉴스레터 한 통을 작성·발송하는
> **marketing-os 의 첫 번째 진짜 에이전트**를 직접 가동한다.
> ⛔ 자동 실행 금지. 단계마다 답을 받은 뒤 다음으로.

## 진행 흐름 (3 단계)

```
1) 소개   이 에이전트가 뭘 하는지 + 연결된 소스          ⏸ 다음
2) 실행   작동 구조 보여주고 실제 실행 (승인 → 발송)      ⏸ y/n 승인
3) 활용   내 업무에 어떻게 적용할지 안내                  ⏸ 선택
```

---

## 1단계 · 소개

이 에이전트가 무엇을 하고, 무엇과 연결되는지 먼저 보여준다.

```
📬 email-newsletter 에이전트

  하는 일 : 매주 지난 주 트렌드를 모아 → 뉴스레터 본문 작성 →
           승인받고 → 메일 발송 → 기록까지 한 번에.

  ⏱ Before 약 1시간 35분/주  →  After 약 1분/주 (승인만 누르면 끝)

🔌 연결된 소스
  ├─ 에이전트  email-newsletter      ← 전체 워크플로를 지휘
  ├─ 스킬      newsletter-writing    ← 본문 800자·제목 5종 작성 (에이전트가 호출)
  └─ MCP 3종
     ├─ Notion   📓  트렌드 읽기 + 아카이브 저장
     ├─ Gmail    📧  뉴스레터 발송
     └─ Discord  💬  승인 요청 + 발송 알림 (선택)

🛡 핵심 : 발송 직전 사람 승인은 절대 생략하지 않는다.
         (tools 에 광고·결제가 없어 돈 나가는 행동은 구조적으로 불가)
```

📐 **이 업무는 스킬일까 에이전트일까?** (5문 판별 — 모든 클립 공통)

```
  Q1 외부 도구(MCP) 필요?   → Gmail·Notion  ✅
  Q2 트리거로 스스로 도나?  → 매주 월 09:00 ✅
  Q3 산출물이 밖으로 나가나? → Gmail·Notion·Discord ✅
  Q4 순수 변환(입력만 바뀜)? → ❌ (밖으로 일함)
  ─────────────────────────────────────────
  → 판정: 🤖 에이전트   (도구·트리거·산출물이 있으면 에이전트)
     단, '본문 작성' 한 토막만은 순수 변환 → newsletter-writing 스킬로 분리 (3-3 제작)
     ⇒ "에이전트 안에 스킬이 산다"의 첫 사례
```

⏸ **"실행 볼까요?" 답 받고 2단계로.**

## 2단계 · 실행

### 2-0. 사전 점검 (조용히)

`claude mcp list` + Notion/Gmail 호출 1회로 연결 확인. ❌ 면 Settings → Connectors
에서 Connect 1클릭. Discord webhook 은 없으면 건너뛴다.

### 2-1. 이 에이전트는 이렇게 만들어졌다 (에이전트 5요소 + 작동 구조)

먼저 `agents/part3-content/email-newsletter.md` 를 펼쳐 **업무 → 에이전트 5요소**를 짚는다 (모든 에이전트 클립 공통 골격):

```
1) frontmatter : name + description     무슨 일을, 언제 하는가
2) tools       : Gmail · Notion · Skill(newsletter-writing)   ← 광고·결제 없음(안전 설계)
3) trigger     : 매주 월 09:00 / /send-newsletter
4) workflow    : 트렌드 → 작성 → [승인 게이트] → 발송 → 기록
5) outputs     : Gmail 1통 · Notion 아카이브 · (Discord)
→ tools·trigger·outputs 가 있으니 스킬이 아니라 '에이전트'. 이 5요소가 .md 한 장의 뼈대다.
```

그리고 그 workflow(④)가 실제로 이 순서로 돈다:

```
🔄 워크플로 (이 순서로 실제 실행됩니다)

  Notion 트렌드 5건
    → newsletter-writing 스킬 호출 (본문 + 제목 5종 + preheader + CTA)
    → ⏸ 승인 게이트 (y/n)
    → 발송 ─┬─ (기본) Gmail 초안 → 사람이 보내기/예약 전송 클릭
           └─ (완전 자동) SMTP 실발송  automation/send_newsletter.py
    → Notion "뉴스레터 아카이브" 1행
    → (webhook 있으면) Discord 발송 확인
```

> ⚠️ **Gmail MCP 는 초안까지만** 지원(발송 API 없음). 그래서 기본은 "초안 → 사람 클릭"
> 이고, 이게 안전 원칙과도 맞는다. 마지막 클릭까지 없애려면 아래 2-4 SMTP 경로를 쓴다.

### 2-2. 데이터 준비 (모드 선택 ⏸)

```
[A] 샘플 모드 ⭐ 추천 — 가상 브랜드 "결로아" 트렌드 5건 자동 시딩
[B] 실데이터 모드 — 기존 "주간 트렌드" DB 사용 (5건 미만이면 샘플 전환)
```

[A] 선택 시 `sample-data/weekly-trends.json` 을 읽어 Notion DB "주간 트렌드"(5행)
+ "뉴스레터 아카이브"(빈 상태) 생성.

### 2-3. 실제 실행

각 단계 결과를 보여주며 진행:

1. Notion 트렌드 5행 로드 → 표
2. newsletter-writing 호출 → 본문 800자 + 제목 5종 미리보기
3. **승인 ⏸ y/n** — y 발송 / n 발송 안 하고 종료 (안전 원칙 시연)
4. Gmail 발송 → Notion 아카이브 1행 → (Discord 알림)

⏸ **승인(y/n) 받은 뒤에만 발송.**

### 2-4. 완전 자동 발송 (선택 · "마지막 클릭까지 없애기")

Gmail MCP 는 초안까지만 만들 수 있어 사람이 보내기를 눌러야 한다. 이 클릭까지 없애려면
**Gmail SMTP + 앱 비밀번호**로 직접 발송한다. (검증된 정기 뉴스레터에만 권장)

```
① 준비 (1회)
   - Gmail 2단계 인증 ON → https://myaccount.google.com/apppasswords 에서 16자리 발급
   - .env 에 등록 (커밋 제외):
       GMAIL_ADDRESS=you@gmail.com
       GMAIL_APP_PASSWORD=abcdefghijklmnop      # 공백 제거 16자
       DISCORD_WEBHOOK_URL=...                   # 발송 후 확인 알림 (선택)

② 발송 (초안 대신 실발송)
   python automation/send_newsletter.py \
     --to <받는사람> --subject "<제목>" \
     --html outputs/{날짜}/email-newsletter/{날짜}-...html
   → smtp.gmail.com:465(SSL) 실발송 → DISCORD_WEBHOOK_URL 로 발송 확인 embed

③ 무인 반복 (Part 10)
   cron/launchd 에 매주 월 09:00 으로 ② 를 걸면 사람 개입 0 의 자동 발송.
```

> 🛡 SMTP 직행은 승인 게이트를 건너뛴다. 신규·일회성은 "초안 + 사람 확인",
> 매주 같은 형식의 정기 뉴스레터만 SMTP 자동 발송으로 분기한다.

## 3단계 · 활용 — 지금, 내 업무로

> 안내로 끝내지 않는다. **바로 "당신 브랜드 버전"을 한 통 만들어보게** 한다.

### 적용 게이트 (능동 질문 ⏸)

```
🚀 그럼, 지금부터 내 업무에 적용해볼까요?

   방금 '결로아' 자리에 당신 브랜드만 넣으면 똑같이 작동합니다.
   아래 3가지만 알려주시면, 지금 바로 당신 브랜드 뉴스레터 1통을 만들어 드릴게요.

   ① 어떤 브랜드/업종인가요?       (예: 홈카페 원두 D2C)
   ② 뉴스레터 독자는 누구인가요?   (예: 30대 직장인 홈카페족)
   ③ 트렌드 소스는 어디서 가져올까요?
        a. 내가 자료 3~5개를 직접 줄게요
        b. 내 업계 뉴스·블로그 URL 몇 개를 줄게요
        c. 일단 샘플로 한 번 더 (결로아)
```

→ 답을 받으면 : 그 입력으로 newsletter-writing 스킬을 **즉석 재호출** → 당신 브랜드
   제목 5종 + 본문 1통 생성 → (원하면) 본인 메일로 초안 발송 → Notion 아카이브.
→ "건너뛰기" 하면 아래 체크리스트만 안내하고 마무리.

### 오래 쓰는 법 (적용 후 정착)

```
1. 트렌드 자동 수집   trend-scanner(4-2) = Firecrawl/WebSearch 로 매주 실제 기사 검색
                     → Notion "주간 트렌드" 적재 (샘플 시딩 불필요, 라이브 수집)
2. 브랜드 톤 입히기   "Brand Voice" 페이지(5-1) → 본문이 내 말투로
3. 완전 무인 가동     매주 월 09:00 cron(Part 10) → `claude` 헤드리스가 검색→작성→발송
                     ※ send_newsletter.py(SMTP)는 발송만, 검색·작성은 Claude(에이전트)
                     ※ 초안+승인 유지하려면 cron 은 초안까지만
4. 구독자 실발송      테스트(1명) → Gmail 라벨 newsletter-subscribers 로 확장
```
