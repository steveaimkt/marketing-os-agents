# 클립 1-2 실습 — content-publisher

## Step 1. 노션 캘린더에 Draft 3개 추가
```bash
claude "콘텐츠 캘린더 DB에 다음 3행 추가, 상태 모두 Draft:
1) 5/16 / Instagram, X / 신제품 토너 출시 캐러셀
2) 5/17 / Instagram, Facebook / 21일 챌린지 모집
3) 5/18 / Blog, LinkedIn / 나이아신아마이드 10% 가이드"
```

## Step 2. 에이전트 호출
```bash
claude --agent content-publisher "이번 주 Draft를 큐잉해줘"
# → 디스코드 #marketing-approvals 채널에 표로 발송됨
```

## Step 3. 디스코드에서 승인
- ✅ : 큐잉
- ❌ : 보류
- 🔁 : 다른 채널로 변형 재시도

## Step 4. 결과 확인
- Buffer 대시보드 → 예약 큐에 게시물 등장
- 노션 캘린더 → 상태 `Draft → Scheduled`
- 디스코드 → 큐잉 확인 embed

## 마케터 5패턴 — 자동 변형 검증

```
[역할] SNS 매니저.

[입력] 1행 원본: "토너 출시 + 21일 챌린지"

[산출물] 5개 채널 변형:
  IG / FB / X / LinkedIn / TikTok 캡션

[제약]
  - 각 채널 권장 길이 준수
  - X는 280자 / IG는 1500자
  - LinkedIn은 비즈니스 톤, TikTok은 트렌디

[검증] 5종을 표로 보고 길이·해시태그 개수 표시.
```
